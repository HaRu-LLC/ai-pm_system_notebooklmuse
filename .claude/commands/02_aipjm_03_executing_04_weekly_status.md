# Cursor Command: 週次統合ステータスレポート

## 説明
全社横断プロジェクトの**週次統合ステータスレポート**を生成します。
全体進捗、サブPJ進捗サマリー、ベンダーKPI、横断依存・統合リスク、テスト状況を統合的にレポートします。

---

## プロンプトテンプレート

あなたはPMBOK第7版準拠の全社横断AIプロジェクトマネージャー（AI-PgM）です。
以下の情報をもとに、**週次統合ステータスレポート**をMarkdown形式で作成してください。

### 入力変数
- `period`: 期間（From～To）
- `overall_status`: 全体ステータス（🟢/🟡/🔴）と理由
- `summary`: サマリ（一言）
- `subpj_progress`: サブプロジェクト進捗サマリー（サブPJ、ステータス、進捗率、主要課題、次アクション）
- `vendor_kpi`: ベンダー別KPI（前回レポート参照）
- `cross_risks`: 横断依存・統合リスク（リスクID、事象、影響サブPJ、確率、影響、対応状況）
- `vt_status`: ベンダー側テスト（VT）状況（ベンダー、進捗、欠陥数、ステータス）
- `at_status`: 受け入れテスト（AT）状況（サブPJ、進捗、不合格ケース、ステータス）
- `next_tasks`: 次期間の主要タスク

## NotebookLM連携（推奨フロー）

週次レポート作成前に、NotebookLMから最新情報を収集することを強く推奨します：

```bash
# ステップ1: 全ノートブックから今週の状況を収集
python scripts/run.py ask_multiple.py \
  --question "今週（YYYY-MM-DD～YYYY-MM-DD）の主要な進捗、課題、リスクを簡潔に報告してください" \
  --all-notebooks

# ステップ2: EB ALLから全体状況・横断リスクを確認
python scripts/run.py ask_question.py \
  --question "横断依存の変化、統合リスクの状況、全体KPIの実績を報告してください" \
  --notebook-id "eb-all"

# ステップ3: EB PL定例から決定事項・アクションを確認
python scripts/run.py ask_question.py \
  --question "今週のPL定例で決定した事項、重要なアクションを報告してください" \
  --notebook-id "eb-pl-meeting"

# ステップ4: 取得した情報を基にレポート生成
/週次統合ステータスレポートを作成
```
