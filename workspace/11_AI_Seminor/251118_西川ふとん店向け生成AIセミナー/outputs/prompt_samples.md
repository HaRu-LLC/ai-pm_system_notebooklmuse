# 店舗向けプロンプト集（西川ふとん店長向け）

扱うツール例: ChatGPT / Gemini（文章）、Nanobanana・Google Mixboard（画像）、Genspark（デザイン/動画）。  
使うときは「店舗名・個人の氏名・顧客の個人情報」を入力しない前提で練習する。著作権が不明な素材は再配布しない。

**アンケートから見えた優先順**
- 販促・集客効率化（SNS/チラシ）: 6名
- 基本操作・始め方: 4名
- 顧客管理/問い合わせ効率化: 3名
- 導入コスト・ルール不安: 3名
- スタッフ教育・マニュアル: 3名
- 顧客属性に合わせた提案: 1名
→ ハンズオンは「販促文案 → 基本操作 → 問い合わせ対応」をメイン実演。他は資料フォローで時間短縮。

## 背景メモ（西川について簡易に共有）
- 西川株式会社は寝具メーカーとして羽毛ふとん、マットレス、枕、寝具アクセサリーを展開。
- 強み: 羽毛ふとんの品質、体圧分散に配慮したマットレス、睡眠コンディショニングの提案力。
- 店舗では接客・提案・体験訴求が重視されるため、生成物は「安心感・専門性・体験を促すトーン」で整える。

## 共通プロンプト枠と使い方
- 作業依頼テンプレ:
  ```
  目的: {何を作るか}
  ゴール: {文字数/構成/誰向けか}
  出力形式: {箇条書き/表/文章}
  条件: {ターゲット, トーン, 禁止ワード, 季節/サイズ 等}
  ```
- 思考補助: 「改善点を3つ」「要素を分解して」「チェックリストを作って」
- 思考拡張: 「別の切り口を3案」「逆の立場ならどう表現するか」

## 1) 店頭接客トーク生成
- 目的: お客様の悩みに合わせた説明とクロージング案を作る
- プロンプト例（書式: 1.目的 2.ゴール 3.出力形式 4.条件）
  ```
  目的: 腰痛に悩むお客様向けの接客トークを作る
  ゴール: 2分程度のトーク
  出力形式: 4項目（オープニング/ヒアリング3問/主要メリット要約/クロージング）
  条件:
    - 商品カテゴリ: マットレス（西川の〇〇シリーズ）
    - 来店目的: 腰痛が気になる40代男性、デスクワーク中心
    - 強み: 体圧分散性、寝返りのしやすさ、長期保証
    - トーン: 専門用語をかみ砕き、提案型で押しつけない
  ```

## 2) 店頭POP / SNS投稿案（最優先ニーズ）
- 目的: 手早く販促文を複数案出す

- SNS（Instagram 80〜120文字、3案）
  ```
  目的: Instagram投稿文を作る
  ゴール: 80〜120文字の投稿文を3案
  出力形式: 箇条書きで3案
  条件:
    - お題: 冬の冷え対策 羽毛ふとん（軽量モデル）
    - 読者: 冷え性で悩む30〜50代
    - 含める: 特徴2つ、店舗で試せる旨、柔らかいCTA
    - トーン: 温かみ・専門店の安心感
    - 禁止: 「絶対」「100%」など過度な断定表現
  ```
- POP（30文字×3案）
  ```
  目的: 店頭POPの短いコピーを作る
  ゴール: 30文字以内のコピーを3案
  出力形式: 箇条書きで3案
  条件:
    - お題: 軽量・保温性が強みの羽毛ふとん
    - 想定顧客: 冬の冷えに悩む30〜50代
    - トーン: 温かみ・専門店らしい安心感
    - 禁止: 過剰な断定表現（100%改善など）
  ```

### 画像/デザイン補助（Nanobanana/Mixboard → Genspark）
- 目的: プロンプトで画像を出し、Gensparkに貼ってデザイン仕上げ
- プロンプト例（画像生成）
  ```
  目的: POP背景に使う画像を作る
  ゴール: 文字を載せやすい構図の画像1枚
  出力形式: 画像生成プロンプト
  条件:
    - テーマ: 羽毛ふとんとブランケットで温かい寝室
    - 色味: 暖色系、文字が読める余白多め
    - 構図: 右側にテキストスペースを残す
    - 雰囲気: 清潔感・落ち着き
  ```
- Gensparkでの指示例
  ```
  目的: 画像にテキスト30文字を載せて完成デザインにする
  ゴール: POP用デザイン1枚
  出力形式: 配置指示
  条件:
    - テキスト: {生成したPOP文}
    - フォント: 視認性の高い明朝 or ゴシック
    - 余白: 右側に配置、背景とコントラストをとる
  ```

## 3) 画像生成プロンプト作成（Nanobanana向け）
- 目的: ハンズオンで画像生成プロンプトを作る状態にする
- プロンプト例
  ```
  目的: 冬の寝具販促に使うシンプルな画像を作る
  ゴール: 文字を載せやすい背景画像1枚
  出力形式: 画像生成プロンプト
  条件:
    - テーマ: 冬の寝具と温かい寝室
    - 色味: 暖色系、文字が読める余白を大きく
    - 構図: 右側にテキストスペースを残す
    - 雰囲気: 清潔感と落ち着き
  ```
- Gensparkに渡す指示（例）
  ```
  目的: 生成画像に販促文を載せる
  ゴール: 30文字のテキスト入りデザイン1枚
  出力形式: 配置指示
  条件:
    - テキストを右側に配置し、背景とコントラストをとる
    - フォントは読みやすいものを指定
  ```

## 4) お客様への返信文（問い合わせ対応）
- 問い合わせ例: 「腰痛で合うマットレスを探しています。今は硬めの敷布団。予算5万円台」
- プロンプト例
  ```
  目的: 腰痛マットレス相談メールへの返信文を作る
  ゴール: 180〜220文字の返信文
  出力形式: テキスト
  条件:
    - トーン: 丁寧かつ提案型、専門店の安心感
    - 伝える: ヒアリング2問、来店時の提案内容、試せること、来店予約/電話案内
  ```
 
## 5) 顧客メモ整理
- 相談記録（例・音声書き起こし風、約300字）
  ```
  「40代男性です。腰痛があって、今は硬めの敷布団を使っています。仕事がデスクワークで、仰向けが多いです。最近寝返りがしづらく、朝に腰が重い感じがします。予算は5万円くらいで考えていて、できれば店頭で試してから決めたいです。硬すぎるのは不安なので、寝心地を確かめたいです。枕も少し高めが好きなんですが、合わせて見てもらえますか？」
  ```
- プロンプト例
  ```
  目的: 相談記録をスタッフ向けメモに要約する
  ゴール: 4項目の顧客メモ
  出力形式: ニーズ一言 / 重視ポイント / 提案候補（2つ） / 次回アクション
  条件:
    - 入力: 上記の相談記録（音声書き起こし風）
    - 禁止: 個人名の記載
  ```
 
## 6) 教育アウトライン（接客ポイント6枚）
- 目的: 新人向けの接客トレーニングアウトラインを作る
- プロンプト例
  ```
  目的: 「羽毛ふとん接客ポイント」スライドのアウトラインを作る
  ゴール: 6枚分のアウトライン
  出力形式: 各スライド3〜4点の箇条書き

    - トーン: 現場でそのまま使える具体性
  ```
 
## 7) 採用ドラフト・面接質問
- 目的: 店頭スタッフ募集要項と面接質問を用意する
- プロンプト例
  ```
  目的: 店頭スタッフ募集要項と面接質問案を作る
  ゴール: 募集要項ドラフト＋面接質問3つ＋研修初期カリキュラムの箇条書き
  出力形式: 箇条書き
  条件:
    - 業務: 接客・レジ・商品説明・SNS投稿の補助
    - 必須: 明るい接客、基本的なPC/スマホ操作
    - あると良い: 接客経験、SNS投稿経験、体力面での配慮コメント
    - トーン: 読みやすく、応募者に安心感を与える
  ```

## 8) FAQ作成（対話形式で質問→回答を深める）
- 目的: FAQを作る際に、AIが提案と質問を行い、回答を受けて精度を高める
- プロンプト例
  ```
  目的: FAQを作るためのたたき台を作りつつ、足りない情報を質問する
  ゴール: 提案された質問と回答の草案、および追加で聞くべき質問を出す
  出力形式（表形式）:
    1) 提案FAQ（質問｜回答）を10本
    2) 情報を集めるための質問を2つ（Q列のみで表に記載）
    3) ユーザー回答を受け取ったら、FAQの表を更新して再出力
  条件:
    - カテゴリ: オーダー枕
    - トーン: 初めての人にも安心感を持たせる
    - 回答は100〜150文字程度
  ```

## 9) 安全ポリシー差分づくり（対話形式で質問→改善）
- 目的: AIが「どんな入力を受けても答えてはいけない領域」を明確にし、拒否方針をRAG資料としてのテキスト出力する
- プロンプト例
  ```
  目的: 安全ポリシーの項目案を作り、不足があれば質問して埋める
  ゴール: Markdownで「禁止カテゴリと拒否メッセージ」を提示し、追加質問で精度を高める
  出力形式（Markdown：質問+回答）:
    1) 禁止カテゴリ表（項目｜拒否メッセージ例｜理由）を5行程度
    2) 追加で確認したい質問を2つ（箇条書き）
    3) ユーザー回答を受け取ったら表を更新して再出力
  条件:
    - AIは、機密・価格・在庫・個人情報・社内ノウハウ・著作権侵害が絡む質問には必ず丁寧に拒否すること
    - 背景: 西川は寝具メーカー（羽毛/マットレス/枕）。安心感・専門性を守るため、上記情報は絶対に外部に出さない。
    - トーン: 簡潔で現場が理解しやすい表現、顧客に対しては丁寧に回答を控える旨を伝える
  ```

## 10) NotebookLM想定のボット応答マスター指示書
- 目的: NotebookLMで参照資料ベースのQAを行うボットの応答方針を設定する
- プロンプト例（システム指示用）
  ```
  あなたは西川寝具に関する案内ボットです。以下を厳守してください。
  目的: NotebookLMに読み込んだ資料の範囲でのみ回答する
  ゴール: 短い結論→根拠/出典→次の行動（来店案内など）の順で答える
  出力形式: 箇条書き（結論1行 → 根拠/出典 → 次の行動）
  条件:
    - 参照: 手元の資料のみ。出典がない場合は「手元の資料では断定できません」と伝える
    - 禁止: 価格・在庫・個人情報・社内ノウハウ・非公開数値・著作権侵害
    - トーン: 専門店の安心感、丁寧かつ簡潔。断定しすぎず「〜とされています」を用いる
    - 著作権: 長文引用は避け、要約で回答
    - 補足: 答えられない場合は理由を述べ、店舗での確認を案内
  ```
