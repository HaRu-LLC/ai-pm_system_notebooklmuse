# 11/18 西川ふとん店向け生成AIセミナー スライド用テキスト

### 0-1 オープニング
- タイトル / 講師紹介
- ゴール: 明日すぐ試す一歩 + 自店向けアイデアを持ち帰る
- 事前アンケート: SNS/チラシ6, 基本操作4, 問い合わせ3, 導入不安3, 教育3, 提案1
- 進行: 基礎→実践①②③→ハンズオン→共有→アクション

### 0-2 生成AIとは / リスク
- 従来: もらったデータを「分類・検索」するAI／生成AI: テキスト・画像を「創り出す」AI（ChatGPT/Gemini/Nanobanana等）
- リスク: 誤情報・幻覚 / 機密・個人情報入力NG / 著作権（素材の扱いに注意）
- 対策: 社外秘を入れない / 出力は必ず目視チェック / 著作権が不明な画像は再配布しない

### 1-1 基礎とプロンプトの基本
- 生成AIの留意: 誤情報, 個人情報/価格表NG, 著作権注意
- 質問の型: 目的 / 対象 / 制約 / トーン / 出力形式
- 5分練習お題: 「羽毛ふとんの特徴を1分で説明」→ 出力を2行だけ修正

### 1-1b プロンプトのお手本
- 目的: 何をしたいか（販促文/返信/要約など）
- ゴール: 成果物の姿（文字数・構成・誰向けか）
- 出力形式: 箇条書き/表/文章、文字数や件数
- 条件: ターゲット/トーン/禁止ワード/季節・サイズなど具体

### 1-1c プロンプトの在り方
- 作業依頼: 「〇〇を作って」「表を出して」→ 具体的な条件を添える
- 思考補助: 「改善点を3つ挙げて」「要素を分解して」→ ブレストやチェック用途
- 思考拡張: 「別の切り口を3案」「逆の立場なら」→ 新しい視点や案出しに使う

### 1-2 ツールの種類
- チャット系: ChatGPT / Gemini（文章・対話、プロンプトで指示）
- 画像・動画・デザイン: Nanobanana / Google Mixboard（画像）, Genspark（デザイン/動画）
- エージェント系: GPTs / Gemini Extensions / GEM（独自指示 + ツール呼び出し + ファイル/URL参照 + Webhook連携）
- 使い分け: 文章→チャット系 / ビジュアル→画像・デザイン系 / 自動化→エージェント系

### 1-3 グループシェア（5分）
- 共有: 「自店でやりたいこと1つ」「気になるツール1つ」
- 目的: 関心の焦点を合わせ、実践パートの着眼点を揃える

### 2-1 実践①概要（SNS/チラシ）
- 成果物イメージ Before/After
- 使うツール: ChatGPT / Gemini（文）| Nanobanana / Google Mixboard（画像）| Genspark（デザイン/動画）
- ポイント: 過剰断定NG / 季節感・サイズ指定 / 柔らかいCTA

### 2-2 テキスト生成ステップ
- 貼り付け→生成→2行修正の3ステップ
- お題: 「冬の冷え対策 羽毛ふとん軽量モデルをInstagramで紹介」
- チェック: 80–120文字 / トーン一致 / CTAあり

### 2-3 POP生成ステップ
- お題: 「軽量・保温性が強みの羽毛ふとん」30文字×3案
- チェック: 過剰断定なし / 30文字厳守 / 温かみ

### 2-4 画像/デザインの流れ（画像プロンプト作成がゴール）
- 画像プロンプトを作る（Nanobanana向け）→ 生成結果をGensparkでデザイン仕上げ
- Mixboardは画像統合ツールとして紹介のみ（必要なら複数画像の合成に使う）
- Genspark側の指示: 生成画像を背景に、テキスト30文字を配置（フォント/余白指定）

### 3-1 実践②概要（問い合わせ・教育・採用）
- 取り組む領域: 返信テンプレ / 顧客メモ整理 / 教育アウトライン / 採用ドラフト
- 使うツール: ChatGPT / Gemini

### 3-2 返信テンプレ
- お題: 「腰痛マットレス相談メールへの返信」
- 出力形式: 返信本文 + 社内メモ
- 修正ポイント: 敬語・価格帯・来店導線・禁止ワード確認

### 3-3 顧客メモ整理
- 入力例: 40代男性/腰痛/硬め布団/予算5万円
- 出力型: ニーズ一言 / 重視ポイント / 提案候補 / 次回アクション

### 3-4 教育・採用・接客練習GPTs
- 接客練習GPTs: よくある質問＋ロールプレイ指示を組み込み、対話で練習する使い方を紹介
- 教育計画: 「羽毛ふとん接客ポイント」6枚アウトラインを例に、研修スケジュールを作る
- 採用: 店頭スタッフ募集要項ドラフト＋面接質問3つ＋研修カリキュラム見本

### 3-5 グループシェア（5分）
- 共有: 「返信テンプレ/顧客メモ/教育/採用のどこを試したいか」＋1つだけ理由
- 目的: ハンズオンで使うお題を決める、優先順位を明確にする

### 4-1 実践③ 概要（西川特化）
- PoC 4ステップ: 1) FAQ整理 2) 安全ポリシー 3) テンプレ運用 4) HP簡易連携
- 店舗HP問い合わせボットの流れ: FAQ→応答→来店誘導
- ツール例: ChatGPT/Gemini API＋FAQ | 画像: Nanobanana/Mixboard | LP: Genspark

### 4-2 PoC手順1: FAQ整理
- 目的: よくある質問を収集し、回答方針を揃える
- 作業: FAQリスト作成・回答のトーン統一・禁止事項明示

### 4-3 PoC手順2: 安全ポリシー
- 目的: 機密・価格・在庫など答えない範囲を明確化
- 作業: NGワード/禁止リスト、個人情報を扱わない指示

### 4-4 PoC手順3: テンプレ運用
- 目的: ボットの応答型を決める
- 作業: 「回答本文＋社内メモ」2段構成テンプレを設定、テストシナリオ10本で検証

### 4-5 PoC手順4: HP連携・運用
- 目的: 店舗HPで問い合わせ応答→来店導線へ
- 作業: 簡易連携の例を提示（完成デモを見せる）、ログ監査と更新フローを説明

### 5-1 ハンズオン案内
- 手順: ログイン→お題プロンプト→生成→自店情報で書き換え→チェック
- チェック: 過剰表現なし / トーン一致 / 行動明確 / 個人情報なし

### 5-2 ハンズオンお題
1) SNS投稿: 「冬の冷え対策 羽毛ふとん軽量モデル」→ ターゲット/商品名を自店版に  
2) 問い合わせ返信: 「腰痛マットレス相談メール」→ 商品名・価格帯・導線を自店版に  
3) 画像/デザイン: 「冬の寝具ビジュアル＋販促テキスト」→ 色味/テキストを自店トーンに  

### 6-1 共有・フィードバック
- グループ: 試したお題 / うまくいった点 / 難しかった点
- 全体: 2例を講師フィードバック（販促文・問い合わせ返信の改善）

### 6-2 グループシェア（5分）
- 共有: ハンズオン成果（1本）と次に直したい点
- 目的: 互いの改善ポイントを学ぶ、次の一歩を具体にする

### 7-1 質疑応答
- 参加者からの質問に回答。時間厳守で主要論点を拾う。

### 7-2 アンケート
- 事後アンケートのURL/QRを提示し、その場で回答を依頼。
