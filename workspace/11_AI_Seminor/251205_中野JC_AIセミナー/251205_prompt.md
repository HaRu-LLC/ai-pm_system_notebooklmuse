## 1. デモ用プロンプト

講師が実演する「中野の魅力再発見プロジェクト」用のプロンプトです。
デモの流れを理解するための参考用としてご活用ください。

### プロンプト1: アイデア出し（デモ用）

```
あなたは{地域活性化プロジェクト}の専門家です。

【背景】
{中野市および山ノ内町}には、{商工会議所青年部、JA青年部、温泉団体、小料理屋組合}など、様々な地域団体が存在します。しかし、団体間の連携が少なく、それぞれが個別に活動している状況です。

【目的】
複数の地域団体が連携し、「{中野の魅力を全国に発信する}」地域活性化事業のアイデアを{5つ}提案してください。

【制約条件】
- 初期予算: {300万円}以内
- 実施期間: {6ヶ月}以内にスタート
- 参加団体: 最低{3団体}以上の連携
- ターゲット: {首都圏在住の30〜50代}
- 季節性: {春〜秋}の観光シーズンを活用

【出力形式】
各アイデアについて、以下の形式で提案してください。
1. プロジェクト名
2. 概要（{100文字}程度）
3. 連携する団体（想定）
4. 期待される効果
5. 初期予算の概算
```

## 3. NotebookLM用プロンプト

### プロンプト5: NotebookLMでプレゼンスライドのデザイン指定

**前提:** Geminiで作成した事業計画書のテキストを、NotebookLMに「ソース」としてアップロード済み。

**操作:** NotebookLMの「ノートブックガイド」から「プレゼンテーション」を選択すると、自動的にスライドが生成されます。

**デザイン指定プロンプト（生成後に調整する場合）:**

```
スライドのデザインを以下のように調整してください。

- トーン: {プロフェッショナル / カジュアル / クリエイティブ}
- メインカラー: {例: ネイビーブルー}
- 1スライドあたりの情報量: {少なめ / 標準 / 多め}
- 強調したい要素: {例: KPI数値を大きく表示}
```

## 4. プロンプトのカスタマイズ方法

### 基本的な書き換えポイント

プロンプトをカスタマイズする際は、以下の【　】部分を自社の情報に書き換えてください。

| 項目 | 書き換え例 |
| :--- | :--- |
| **業界名** | 飲食業、製造業、観光業、小売業、サービス業、農業 |
| **事業内容** | 居酒屋を経営、農産物を生産、温泉旅館を運営、アパレル店を経営 |
| **現在の課題** | 売上の季節変動、新規顧客獲得、人手不足、ブランド認知度が低い |
| **目的** | 新規事業の立ち上げ、既存事業の拡大、地域貢献、売上向上 |
| **予算** | 50万円、100万円、200万円（実情に合わせて） |
| **期間** | 1ヶ月、3ヶ月、6ヶ月（実情に合わせて） |
| **ターゲット** | 30代女性、地元住民、観光客、オフィスワーカー |

### 追加指示のテクニック

プロンプトを実行した後、以下のような追加指示でさらに改善できます：

```
「もっと具体的に数値を入れてください」
「若い世代向けのトーンに変えてください」
「箇条書きを表形式に変換してください」
「デメリットも含めて分析してください」
「競合との比較表を作成してください」
「実行ステップを週単位で細分化してください」
```

## 5. よくあるトラブルと対処法

### トラブル1: プロンプトを入れても期待した回答が返ってこない

**原因:**
- プロンプトが抽象的すぎる
- 制約条件が不足している

**対処法:**
```
「もっと具体的にお願いします。特に【具体化したい項目】について詳しく教えてください」
```

### トラブル2: 出力が長すぎる/短すぎる

**対処法:**
```
「もっと簡潔にまとめてください。各セクション200文字以内で」
「もっと詳細に記述してください。各セクション500文字程度で」
```

### トラブル3: 自社の業界に合わない回答が返ってくる

**対処法:**
- プロンプトの冒頭の「役割」をより具体的に指定
```
悪い例: 「あなたは経営コンサルタントです」
良い例: 「あなたは中小飲食店の経営改善を専門とするコンサルタントです」
```

### トラブル4: NotebookLMにアップロードできない

**原因:**
- ファイル形式が非対応
- ファイルサイズが大きすぎる

**対処法:**
1. テキストをコピー
2. NotebookLMの「テキストを貼り付け」機能を使用
3. または、Googleドキュメントに保存してからアップロード

### トラブル5: Geminiの応答が途中で止まる

**原因:**
- 出力が長すぎて途中で切れた

**対処法:**
```
「続きをお願いします」
```
と入力すれば、続きを出力してくれます。
