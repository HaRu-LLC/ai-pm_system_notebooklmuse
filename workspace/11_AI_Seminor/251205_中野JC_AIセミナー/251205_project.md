# 中野JC向け生成AIセミナー 企画書 ver.2.0

## 1. プロジェクト概要

- **セミナー名:** 経営者の武器になるAI活用術 ～事業計画書を30分で作る実践セミナー～
- **開催日:** 2025年12月5日（金）16:10〜18:10
- **対象:** 中野JC、商工会議所青年部、JA青年部、温泉団体、小料理屋組合、その他地域団体
- **時間:** 100分（セミナー25分＋ワークショップ45分＋発表・講評20分＋質疑10分）
- **会場:** 信州中野商工会議所 2階大会議室
- **主催:** 公益社団法人中野青年会議所 会員フォローアップ特別室
- **講師:** 井戸聞多（合同会社HaRu 代表社員）

---

## 2. 中野JCの事業目的との整合性

### 事業テーマ
**「広がる輪、輝く未来」**

### 中野JCが掲げる目的
1. 中野市および山ノ内町の同世代の若者が団体を超えて交流し、新たなつながりを築く
2. 地域の団体を知ることで地域の人を知り、会員拡大の可能性を広げる
3. 次年度に向け、予定者を含めたメンバー同士の親睦および地域への理解を深める

### 本セミナーの貢献
- **異業種交流の促進:** ワークショップで異業種混合グループを編成し、団体の垣根を越えた交流を実現
- **地域理解の深化:** デモで「中野の魅力再発見プロジェクト」を題材にし、地域連携の重要性を体感
- **実践的スキル習得:** AI活用という共通言語を得ることで、参加者間の継続的な情報交換・協力関係を構築
- **第2部（懇親会）への接続:** セミナーで形成されたグループが懇親会でも交流を続ける仕掛け

---

## 3. セミナーの戦略的目標

### ミッション
参加者が生成AIを「単なる便利ツール」ではなく「事業を加速させるパートナー」として認識し、たった30分で実用的な事業計画書とプレゼン資料を作成できることを体験することで、明日からの業務に即座に活用できる自信を持って帰ってもらうこと。

### 戦略
1. **パラダイムシフト:** 「AI = テキスト生成」→「AI = 思考の壁打ち相手＋資料作成の自動化」
2. **圧倒的な成功体験:** 45分のワークで「事業計画書＋プレゼンスライド」を完成させる体験
3. **地域連携の視点:** デモで地域団体の協働事例を扱い、JCの事業テーマと接続
4. **即時性:** セミナーで学んだプロンプトをそのまま持ち帰り、翌日から使える
5. **競争と表彰:** 優秀賞（3名）の設定で、ワークへの本気度を高める

### 5.1. オープニング（10分）16:10〜16:20

**進行:**

#### Geminiとの対話デモ（2分）16:16〜16:18

**目的:**
- 生成AIの実力を冒頭で体感してもらう
- 参加者の心を掴み、セミナーへの期待感を高める
- AI = 難しい技術ではなく、「話しかける相手」という認識を持ってもらう

**使用ツール:**
- Google AI Studio（Gemini 2.0 Flash with Live API）
- Webカメラストリーミング機能

**デモの流れ:**

1. **セットアップ（15秒）**
   - 講師がノートPCの画面をプロジェクターに投影
   - Google AI Studioを起動
   - Webカメラで会場を映す

2. **Geminiとの対話（1分30秒）**

   **講師:**
   「Gemini、今日はよろしくお願いします。会場の様子はどうですか？」

   **Gemini（想定応答）:**
   「会場には30名ほどの方がいらっしゃいますね。皆さん真剣な表情で、これからのセミナーに期待されているようです。中野JCの皆さん、本日はよろしくお願いいたします！」

   **講師:**
   「今日は、皆さんに事業計画書を作ってもらいますが、何かアドバイスはありますか？」

   **Gemini（想定応答）:**
   「事業計画書は、アイデアを形にする第一歩です。今日は私が皆さんのアイデア出しから、計画の深掘りまでお手伝いします。遠慮なく、どんどん質問してくださいね！」

3. **講師からのつかみ（15秒）**
   「皆さん、今見ていただいた通り、Geminiは会場の様子を見て、リアルタイムで応答できます。今日、皆さんの事業に月額数千円で雇える超優秀なアシスタントが入社します。この新しいパートナーを使って、たった30分で事業計画書とプレゼン資料を作る方法をお教えします。」

**技術的準備:**
- Google AI Studioアカウント（事前ログイン確認）
- ノートPCのWebカメラ動作確認
- 会場のインターネット環境テスト（帯域幅・安定性）
- プロジェクターへの画面投影設定
- 音声の聞こえやすさ確認（スピーカー音量調整）

**リハーサル:**
- セミナー前日または当日午前中にリハーサル実施
- Geminiの応答速度・内容の確認
- 会場の照明・カメラアングルの調整

**フォールバック（障害時の代替案）:**
万が一、Webカメラストリーミングが技術的に動作しない場合：
- 通常の講師挨拶に即座に切り替え
- 「本来ここでGeminiとの対話デモを予定していましたが、技術的な問題で後ほど実施します」とアナウンス
- セミナー本編のGeminiデモで追加の時間を取り、挽回する

**期待される効果:**
- 参加者が「AIってこんなに自然に会話できるんだ」と驚く
- セミナーへの期待感が一気に高まる
- 堅苦しくなりがちな公開例会の冒頭を、インパクトのある演出で盛り上げる
- SNSでの拡散ネタになる（「Geminiが挨拶するセミナーに参加した！」）

### 5.2. 第1部：セミナー（25分）16:20〜16:45

| 時間 | 内容 | 目的 |

#### 5.2.1. 生成AIとは・リスク（5分）

**トピック:**
- 生成AIの定義（テキスト、画像、音楽などを「生成」するAI）
- 主要ツール：ChatGPT、Gemini、Claude、Copilot
- Gemini選定理由：Google連携、無料でも高機能、NotebookLMとの相性

**リスク＆対策:**
| リスク | 対策 |
| :--- | :--- |
| 情報漏洩 | 機密情報・個人情報は絶対に入力しない |
| 誤情報 | 出力は必ずファクトチェックする |
| 著作権 | 商用利用前に利用規約を確認 |

#### 5.2.2. プロンプトの基本（5分）

**プロンプトの「型」:**
```
【役割】あなたは〇〇の専門家です
【背景・目的】〇〇を実現したい
【制約】予算・期間・対象
【出力形式】箇条書き・表・段階的に
```

**悪い例 vs 良い例:**

**悪い例:**
```
事業計画書を作って
```

**良い例:**
```
あなたは経営コンサルタントです。
私は飲食店を経営しており、新規事業として
テイクアウト専門店を立ち上げたいと考えています。

以下の形式で事業計画書の構成案を作成してください。
- 事業概要
- 市場分析
- ターゲット顧客
- 収支計画
- リスクと対策
```

#### 5.2.3. Geminiで事業計画書作成デモ（15分）

**デモテーマ:**
「中野の魅力再発見プロジェクト」
複数の地域団体が連携し、中野市の魅力を全国に発信する地域活性化事業

**デモの流れ（配布プロンプト使用）:**

**ステップ1: アイデア出し（3分）**
```
プロンプト1を使用
→ 5つのアイデアが出力される
→ 画面を見せながら「このように短時間で多様なアイデアが出ます」
```

**ステップ2: アイデアの選定（3分）**
```
プロンプト2を使用
→ 各アイデアの評価（実現可能性、インパクト、予算）
→ 「このように評価軸を与えると、客観的に判断できます」
```

**ステップ3: 選んだアイデアの深掘り（4分）**
```
プロンプト3を使用
→ 詳細な事業計画書の構成案が生成される
→ 「ここまで来れば、あとは肉付けするだけです」
```

**ステップ4: 事業計画書の完成（5分）**
```
プロンプト4を使用
→ 各セクションの詳細が埋まっていく様子を見せる
→ 「これで事業計画書のドラフトが完成しました」
```

**デモのポイント:**
- プロンプトを順番に実行し、思考が深まっていく様子を見せる
- 「地域連携」をテーマにすることで、JCの事業目的と接続
- 参加者が「自分もできそう」と思える難易度設定

### 5.3. 第2部：ワークショップ（45分）16:45〜17:30

#### 目的
参加者が自分の事業（会社・団体活動）について、Gemini＋NotebookLMを使って、事業計画書とプレゼン資料を実際に作成する。

#### 準備物
- 参加者：ノートPC、Googleアカウント
- 配布物：プロンプト集（251205_prompt.md）、操作手順書
- Wi-Fi環境：全員が同時アクセスできる帯域

#### ワークショップの流れ

##### Phase 1: Geminiで事業計画書作成（30分）16:45〜17:15

**作業内容:**
参加者は配布されたプロンプト集を使い、自分の事業について以下を作成：

1. **アイデア出し（5分）**
   - プロンプト1を自社用にカスタマイズして実行
   - 5つのアイデアを出力

2. **アイデア選定（5分）**
   - プロンプト2で各アイデアを評価
   - 1つに絞る

3. **詳細化（10分）**
   - プロンプト3で構成案を作成
   - プロンプト4で各セクションを充実させる

4. **仕上げ（10分）**
   - 必要に応じて追加質問・修正
   - テキストをコピーしてメモ帳やGoogleドキュメントに保存

**講師のサポート:**
- 巡回しながら困っている参加者をサポート
- よくある質問をその場で全体共有

##### Phase 2: NotebookLMデモ＋各自スライド作成（15分）17:15〜17:30

**NotebookLMデモ（5分）17:15〜17:20**

講師が画面共有で実演：
1. NotebookLM（notebooklm.google.com）にアクセス
2. 「新しいノートブック」を作成
3. 作成した事業計画書のテキストをアップロード（ソースとして追加）
4. プロンプト5を使ってプレゼンスライドの構成案を生成
5. 「ノートブックガイド」から「ブリーフィングドキュメント」を生成
6. 生成されたスライド案を確認

**各自スライド作成（10分）17:20〜17:30**

参加者が実際に操作：
1. NotebookLMに自分の事業計画書をアップロード
2. プロンプト5でスライド構成案を生成
3. 内容を確認・必要に応じて修正
4. グループ発表用に準備

**サポート体制:**
- 技術サポート担当（1〜2名）が巡回
- ログインできない、アップロードできない等のトラブル対応

### 5.6. 質疑応答（10分）17:50〜18:00

**想定される質問と回答:**

**Q1: 無料版と有料版、どちらを使うべき？**
A: まずは無料版で十分。Geminiの無料版でも今日のワークは全てできます。毎日使うようになったら有料版（月2,900円）を検討してください。

**Q2: NotebookLMで作ったスライドはどう活用する？**
A: そのままプレゼンに使うこともできますし、PowerPointやGoogleスライドにコピーして装飾することもできます。あくまで「叩き台」として活用するのがコツです。

**Q3: 機密情報を扱う場合は？**
A: Googleの法人向けプラン（Google Workspace）なら、データが学習に使われない設定が可能です。重要な情報は匿名化・抽象化してから入力してください。

**Q4: 今日のプロンプトは他の用途にも使える？**
A: はい。「事業計画書」を「マーケティング計画」「採用計画」「イベント企画書」などに置き換えれば、幅広く応用できます。
