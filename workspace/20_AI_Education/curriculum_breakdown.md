# AIポリテラシー研修 カリキュラム分解 v0.1

## 1. 全体像（Lead Engineer・Customer Success協議）
- 学習構造: Phase1〜4を13セッション想定（1セッション60分）。
- 進行リズム: 週1回＋宿題（30〜60分）。Phaseごとに成果物レビュー会を設定。
- ファシリテーション: 主担当ファシリ1名、補助1名。演習は2〜3名グループで実施。

## 2. セッション別アウトライン

### Phase 1 Handcrafted Prompting（基礎固め）
- **Session 1: プロンプト基礎構造**
  - 目的: 目的・前提・出力指定の分解を理解。
  - 講義内容: ビジネス課題の三要素、チャットAIが誤読するポイント、よくある失敗例（指示なし・前提欠落・出力曖昧）。
  - 演習内容: 
    - グループで提供課題を分解し、骨組みシートに記入。
    - サンプルプロンプトの構造を色分けし、抜け要素を議論。
  - 成果物: プロンプト骨組みシート。
  - 事前準備: 参加者の課題棚卸メモ（10分）。
- **Session 2: 表現磨きと明確化**
  - 目的: 曖昧表現の洗い出しと改善。
  - 講義内容: 曖昧語リスト、明確化の5ステップ（誰・何を・いつ・どれくらい・評価軸）。
  - 演習内容: 
    - 曖昧語→明確語変換ワーク（タイムボックス10分）。
    - 業界別ケースで明確化を行い、ペアで相互添削。
  - 成果物: Before/Afterログ、改善メモ。
- **Session 3: 改善ループ導入**
  - 目的: 「不明点→回答→改善案」ループを使いこなす。
  - 講義内容: ループの狙い、AIへの要求方法、終了条件の設定例。
  - 演習内容: 
    - 講師デモ後に、ペアでループを実行しログを記録。
    - 個別で自分の課題に適用し、ループの改善点を整理。
  - 成果物: 改善ループ記録、AIへの指示テンプレ。

### Phase 2 Co-Creative Prompting（共創）
- **Session 4: AIと作るプロンプト原案**
  - 目的: 「こんなプロンプトを作りたい」とAIに伝え、原案を共創する手順を習得。
  - 講義内容: メタプロンプトの構造、AIに任せる範囲と人が決める範囲、原案からの差分確認ポイント。
  - 演習内容: 
    - チームでメタプロンプトを書き、AIに原案を生成させる。
    - 原案に対して不足・過剰・誤解ポイントをチェックリストで確認。
    - 原案を元にAIと改善ループを1回実施し、完成版を導出。
  - 成果物: 共創プロンプト原案と改善ログ。
- **Session 5: AIとの磨き込み（ロール＋ガードレール）**
  - 目的: AIと対話しながら役割付与・視点追加・ガードレール設計を同時に行う。
  - 講義内容: 視点提示の順序、AIがロールを誤認した際の修正例、リスク分類と制約文の書き方。
  - 演習内容: 
    - AIにロール案とガードレール案を提案させ、採用/調整判断を記録。
    - 仕上がったプロンプトをテストし、評価シートで質を確認。
  - 成果物: 最終プロンプト、ロール＆ガードレール評価シート。

-### Phase 3 Custom Instructions & Knowledge Base（構築）
- **Session 6: 知識ベース素材ブリーフ作成**
  - 目的: どの業務資料をAIに渡すかを選定し、取り込み方針を決める。
  - 講義内容: ソース選定基準、機密区分、メタデータ設計、Markdown整形の原則。
  - 演習内容: 
    - 現行資料を棚卸し、活用スコアをつける。
    - 取り込み優先順位と整形TODOをブリーフにまとめる。
  - 成果物: 知識ベース素材ブリーフ。
  - 次回準備: ブリーフ要点を整理し、`Meta Prompt Builder`へ渡すためのメモを作成。
- **Session 7: カスタム指示テンプレ徹底演習**
  - 目的: GPTs用指示の骨子を完成度高く作成。
  - 講義内容: 指示テンプレの構造（役割・ゴール・入力形式・禁止事項）、成功例の分解、マルチ業界の比較、`Meta Prompt Builder`と`Instruction Reviewer`を用いたドラフト→検証フロー。
  - 演習内容: 
    - `Meta Prompt Builder`で生成した初稿をテンプレへ転記し、自分の言葉で整える。
    - `Instruction Reviewer`で欠落・曖昧点を洗い出し、ピアレビューと合わせて改善案を確定。
  - 成果物: 指示テンプレ完成版とInstruction Log。
- **Session 8: AI共創ドキュメント構築**
  - 目的: AIと対話しながら要件定義書などのドキュメントをゼロから共創する手順を学ぶ。
  - 講義内容: メタプロンプト事例、アウトライン→詳細化のステップ、AI提案の評価基準。
  - 演習内容: 
    - 共通テンプレートをAIに依頼し、ドラフトを生成。
    - ドラフトを章ごとにAIと議論し、完成度を高める。
  - 成果物: 共創ドキュメント（要件定義書など）の完成版と改善ログ。
- **Session 9: 既存資料リファイン**
  - 目的: 既存メモ・議事録をMarkdown化し、知識ベース向けに再構造化する。
  - 講義内容: Markdown整形のベストプラクティス、メタデータ付与、重複解消テクニック。
  - 演習内容: 
    - 既存資料を整理し、AIに要約やタグ付けをサポートさせる。
    - Googleスプレッドシートでメタデータ表を作成し、レビューループを回す。
  - 成果物: リファイン済みMarkdownファイル、メタデータリスト（ドラフト）。
- **Session 10: 知識ベース統合と検証**
  - 目的: 共創ドキュメント・既存資料を統合し、GPTsに投入→動作検証まで行う。
  - 講義内容: アップロード手順、差分更新、ログ管理、検証観点。
  - 演習内容: 
    - ファイルを投入し、リンク・参照・応答を確認。
    - シナリオテストを実行し、改善プランを策定。
  - 成果物: メタデータリスト最終版、投入チェックリスト、テストログ。

-### Phase 4 Workflow Alignment（業務解体＋ジェム実装）
- **Session 11: 業務観察と分解**
  - 目的: 業務フローを観察しAI化ポイントを特定。
  - 講義内容: 業務観察の視点（工数・品質・リスク）、バリューストリームの描き方。
  - 演習内容: 
    - 自部署の業務を書き出し、非効率セグメントをマーク。
    - グループで難所を共有し、AI化候補を議論。
  - 成果物: 業務解体シート。
- **Session 12: 改善シナリオ設計**
  - 目的: 提供ワークフローとの結合を設計。
  - 講義内容: 提供ワークフロー例の読み解き方、ジェム設計のポイント、依存関係の整理。
  - 演習内容: 
    - ケースマッピングで現状フロー→改善フローを描く。
    - 必要なジェム要件を洗い出し、優先順位を決定。
  - 成果物: AI活用ロードマップ。
- **Session 13: ジェム実装＆レビュー**
  - 目的: GPTsジェムを組み込み、効果を検証。
  - 講義内容: 実装チェックポイント、テスト観点、ROI算出方法。
  - 演習内容: 
    - ジェムを提供ワークフローに接続し動作確認。
    - ペアレビューで改善点を抽出し、ROI試算の数字を整理。
  - 成果物: ジェム仕様書、検証レポート。

## 3. 業界別演習ケース（Customer Success・Industry Advisors協議）
- 製造業
  - ケース1: 不良分析レポートの要約自動化。
  - ケース2: 作業標準書の更新支援。
  - 想定リスク: 法規制対応、機密情報扱い。
- サービス業（宿泊・飲食）
  - ケース1: 口コミ対応と改善提案。
  - ケース2: 週次スタッフブリーフィング資料作成。
  - 想定リスク: 顧客データ匿名化、トーン＆マナー。
- 専門職（士業・コンサル）
  - ケース1: クライアント報告書の骨子作成。
  - ケース2: 調査メモの要約と論点整理。
  - 想定リスク: 守秘義務、証拠確認。

## 4. 宿題・補助教材
- 宿題パターン
  - Phase1: プロンプト改善ログ（Googleドキュメント）、曖昧語変換ワークシート（Googleスプレッドシート）。
  - Phase2: 質問カタログ更新（Googleドキュメント）、ロール＆ガードレール評価シート（Googleスライド）。
  - Phase3: 知識ベース素材ブリーフ（Googleドキュメント）、GPTs指示テンプレ更新（Googleドキュメント）、知識ベースメタデータ表（Googleスプレッドシート）。
  - Phase4: 業務観察ジャーナル（Googleドキュメント）、AI活用ロードマップ（Googleスライドテンプレ）、ジェム検証ログ（Googleスプレッドシート）。
- 補助教材・ツール（ChatGPT Teams前提）
  - `Prompt Skeleton Sheet`（Googleスプレッドシート）: 基本構造を記入。**共有場所:** Teams内共有ドライブ/Google共有ドライブ。
  - `Ambiguity Deck`（Googleスライド）: 曖昧表現カード。**セッション**: Phase1 Session3。
  - `Prompt Refinement GPT`（チャットGPT GPTs）: プロンプト改善ループを支援するテンプレGPT。Session4で使用。
  - `Meta Prompt Builder`（GPTs）: メタプロンプト生成専用ツール。Session5で配布。
  - `Role & Guardrail Board`（Googleスライド）: 視点カード配置。**セッション**: Phase2 Session6。
  - `Guardrail Checker GPT`（GPTs）: 制約文のチェック自動化。Session6宿題で活用。
  - `Custom Instruction Template`（Googleドキュメント）: GPTs指示の雛形。Session8で共有。
  - `Instruction Reviewer GPT`（GPTs）: 指示テンプレのレビュー支援。Session8演習で利用。
  - `Knowledge Base Planner`（Googleスプレッドシート）: ファイル命名・更新ルール。Session9で使用。
  - `Doc Co-Author GPT`（GPTs）: 要件定義等を共創するGPT。Session9で活用。
  - `Markdown Refiner GPT`（GPTs）: 既存資料の整形支援。Session10で使用。
  - `Scenario Test Log`（Googleスプレッドシート）: テスト記録テンプレ。Session11で入力。
  - `Workflow Observation Sheet`（Googleスプレッドシート）: 業務観察。Session12で利用。
  - `Gem Spec Template`（Googleドキュメント）: ジェム仕様書。Session14成果物。
  - `ROI Calculator`（Googleスプレッドシート）: 効果計算。Session14で導入。
  - `Team Workspace`（ChatGPT Teamsプロジェクト）: 上記GPTsをまとめて共有、バージョン管理するためのリポジトリ。
- デモ用サンプル一覧（準備推奨）
- Phase1 Session1: マインドセット比較チャート（部下タイプ / 思考パートナータイプ）、ケースディスカッション用メモ。
- Phase1 Session2: 共通課題例（製造・サービス・専門職）各1本、良い/悪いプロンプトの対比サンプル。
- Phase1 Session3: 曖昧語リストと改善例、業界別ステートメントのサンプル（CSV）。
- Phase1 Session4: 改善ループ実行ログの例（スクリーンショット＋テキスト）。
- Phase2 Session5: メタプロンプト例とAI生成原案、改善チェックリストの記入例。
- Phase2 Session6: ロール提案サンプル、ガードレール誤設定例と修正案。
- Phase3 Session7: 素材棚卸し用のサンプル資料（議事録、FAQ、マニュアルの抜粋）。
- Phase3 Session8: 優秀なカスタム指示例（匿名化済み）、評価コメントサンプル。
- Phase3 Session9: AI共創ドキュメントのドラフト→完成版例、対話ログ抜粋。
- Phase3 Session10: Markdown整形前後の比較ファイル、メタデータ表の書式例。
- Phase3 Session11: アップロード後の挙動テスト結果（成功/失敗）、ログ記録サンプル。
- Phase4 Session12: 業務フローマップのサンプル（現状 vs 改善案）。
- Phase4 Session13: AI活用ロードマップの完成例、ジェム要件リスト例。
- Phase4 Session14: ジェム仕様書サンプル、ROI計算書の記入例。
