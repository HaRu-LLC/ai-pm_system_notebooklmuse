# 参考アウトライン集

**作成日:** YYYY-MM-DD
**用途:** AI共創ドキュメント作成時の参考

---

## 使い方

この集は、Session8でAIと共創する際に参考にできるアウトライン例をまとめたものです。

### 活用方法
1. 作成するドキュメントの種類に近いアウトラインを選択
2. 章立てを参考に自分のドキュメントのアウトラインを設計
3. 設計チャットでAIに「以下のアウトラインを参考にして」と伝える
4. 自分の業務に合わせてカスタマイズ

## 1. オンボーディング要件定義書

### アウトライン

```markdown
## 1. エグゼクティブサマリー
   - 目的
   - 主要な提案
   - 次のアクション

## 2. 背景と課題
   - 現状
   - 課題
   - 課題による影響

## 3. 目的とゴール
   - 目的
   - 定量ゴール（KPI）
   - 定性ゴール
   - 成功条件

## 4. スコープと前提
   - 対象顧客セグメント（SMB / Mid-Market / Enterprise）
   - 対象フェーズ
   - 除外範囲
   - 前提条件

## 5. オンボーディング施策
   - 施策一覧（表形式）
   - 施策詳細（各施策ごと）
     - 目的
     - 対象
     - 手順
     - 成果物
     - 担当
     - 期限

## 6. KPIとモニタリング
   - KPI定義（30日定着率、初回サクセスコール完了率など）
   - ダッシュボード
   - レビュー会議

## 7. リスクと対策
   - リスク一覧（表形式）
   - リスク詳細（各リスクごと）

## 8. TODOと次のアクション
   - 短期（1ヶ月以内）
   - 中期（3ヶ月以内）
   - 長期（6ヶ月以内）

## 9. 付録・参考資料
   - 参照ドキュメント
   - 用語集
   - 変更履歴
```

## 3. ナレッジベース記事（FAQ）

### アウトライン

```markdown

   - この記事の目的
   - 対象読者

## 2. FAQ一覧（表形式）
   | No | カテゴリ | 質問 | 回答 |
   |----|---------|------|------|
   | 1 | オンボーディング | 期間は？ | 通常30日です |
   | 2 | サクセスコール | 頻度は？ | 週1回です |

## 3. カテゴリ別詳細
   ### オンボーディング
   #### Q1: [質問]
   **A:** [回答]
   **参照:** [リンク]

   ### 活用促進
   #### Q2: [質問]

   **参照:** [リンク]

## 4. トラブルシューティング

   - 解決手順
   - エスカレーション先

## 5. 関連資料
   - リンク集
   - 動画チュートリアル

## 6. 更新履歴
```

## 5. サクセスコールチェックリスト

### アウトライン

```markdown
## 1. 事前準備
   - [ ] 顧客情報確認（契約プラン、契約開始日）
   - [ ] 利用状況確認（ログイン頻度、主要機能の利用）
   - [ ] ヘルススコア確認
   - [ ] 前回コールの議事録確認

## 2. コール冒頭（5分）
   - [ ] アイスブレイク
   - [ ] 今日のゴール共有
   - [ ] 前回のフォローアップ

## 3. 現状確認（10分）
   - [ ] 利用状況ヒアリング
   - [ ] 成果の確認（KPI達成状況）
   - [ ] 課題のヒアリング

## 4. 提案・支援（10分）
   - [ ] 活用Tips提案
   - [ ] 追加機能紹介
   - [ ] トレーニング提案

## 5. ネクストステップ（5分）
   - [ ] アクションアイテム確認
   - [ ] 次回コール日程調整
   - [ ] 緊急連絡先確認

## 6. コール後
   - [ ] 議事録作成（CRMに記録）
   - [ ] アクションアイテムをタスク化
   - [ ] ヘルススコア更新
```

## アウトライン選択ガイド

| ドキュメント種別 | 推奨アウトライン | 主な用途 |
|---------------|----------------|---------|
| 業務要件定義 | オンボーディング要件定義書 | 新規プロセス設計 |
| 運用マニュアル | サクセスプレイブック | 日常業務の標準化 |
| FAQ・ナレッジ | ナレッジベース記事 | 情報共有・検索 |
| 会議資料 | 定例会議アジェンダ | 定例報告・意思決定 |
| チェックリスト | サクセスコールチェックリスト | 品質担保・抜け漏れ防止 |
| 対応手順書 | エスカレーション対応ガイド | インシデント対応 |
| プロジェクト管理 | プロジェクト計画書 | プロジェクト推進 |

## カスタマイズのヒント

### 1. 章の追加・削除

不要な章は削除し、必要な章を追加してください。

**追加例:**
- コスト・予算
- セキュリティ・コンプライアンス
- トレーニング計画

### 2. 表形式の活用

情報を整理しやすくするため、表形式を積極的に活用してください。

### 3. 視覚化

複雑なフローや構造は、Miro/FigJam/Googleスライドで図解してください。
