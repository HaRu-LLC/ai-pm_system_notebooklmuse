# Part 1: このプログラムで目指すこと

---

# プログラムのゴール

## 「メタプロンプト」で業務GPTsを構築・運用できる人材になる

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   手書きプロンプト → メタプロンプト → GPTs構築 → 業務運用│
│                                                         │
│   Phase 1-2          Phase 2-3        Phase 3-4         │
│   （基礎習得）        （AI共創）      （実運用）         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

---

# 「メタプロンプト」とは？

### プロンプトを生成するプロンプト

```
【従来のAI活用】
毎回、一からプロンプトを考える → 品質バラバラ、属人化

【メタプロンプト活用】
1つのメタプロンプト → 目的を入力 → 最適なプロンプトが生成される
```

**本プログラムの核心**:
- Phase 1-2でメタプロンプトを作れるようになる
- Phase 3-4でメタプロンプトを使ってGPTsを構築・運用する

---

# なぜ「メタプロンプト」が重要なのか

| 課題 | メタプロンプトで解決 |
|---|---|
| 毎回プロンプトを考える | 目的を伝えるだけで自動生成 |
| 人によって品質がバラバラ | 構造化されたプロンプトで品質安定 |
| ノウハウが属人化 | テンプレート化でチーム共有 |
| GPTs設計が難しい | AIと対話しながら設計書を生成 |

**→ 「すべてはメタプロンプトから始まる」**

---

# 3タイプの受講者に対応

| タイプ | 現状 | 本プログラム後 |
|---|---|---|
| **A: 積極利用者** | 日常的にAI活用 | メタプロンプト設計者へ |
| **B: これから使う人** | 興味はあるが未活用 | 業務でAIを自在に活用 |
| **C: ゼロスタート** | ほぼ未経験 | 自信を持ってAIを使える |

**どのレベルからでも、確実にステップアップ**

# Phase 1: 手書きプロンプト基礎（Week 1-3）

### 学ぶこと

| Session | テーマ | 身につくスキル |
|---|---|---|
| 1 | プロンプトの4要素 | 指示・文脈・制約・出力形式 |
| 2 | 文脈・制約・出力形式 | ペルソナ設定、業務プロンプト |
| 3 | 対話型プロンプト | 改善ループ、要件定義→実行 |

**覚える合言葉**: 「し・ぶ・せ・だ」（指示・文脈・制約・出力形式）

---

# Phase 2: メタプロンプト設計（Week 4-5）

### 学ぶこと

| Session | テーマ | 身につくスキル |
|---|---|---|
| 4 | 複合プロンプト | 改善ループ＋要件定義→実行の組み合わせ |
| 5 | メタプロンプト | プロンプトを生成するプロンプト |

**ここが本プログラムの転換点**:
- 「AIにプロンプトを書いてもらう」体験
- 以降、すべてメタプロンプトで進める

---

# Phase 3: GPTs構築（Week 6-10）

### 学ぶこと

| Session | テーマ | 身につくスキル |
|---|---|---|
| 6 | GPTsの概念と企画 | メタプロンプトでGPTs企画書生成 |
| 7 | Instructions設計 | メタプロンプトでInstructions生成 |
| 8 | 知識ベース構築 | メタプロンプトで知識ファイル設計 |
| 9 | テストと改善 | メタプロンプトでテストケース生成 |
| 10 | 仕様書完成 | メタプロンプトで仕様書統合 |

**すべてをAIと共創**: 手書きではなくメタプロンプトで生成

---

# Phase 4: 業務GPTs実運用（Week 11-13）

### 学ぶこと

| Session | テーマ | 身につくスキル |
|---|---|---|
| 11 | 業務プロセス分析 | メタプロンプトでフロー図・ボトルネック分析 |
| 12-13 | 最終課題（2週間） | 設計→実装→運用→20分発表 |

**最終課題の要件**:
- 自社業務でGPTsを構築
- 5回以上使用して運用ログを記録
- フィードバック→改善を実施
- 20分の最終発表

---

| **4** | **業務GPTs＋発表資料** | **実際に使えるツール** |

**最終的に「自分の業務で使えるGPTs」を作る**

# Part 3: 学習の進め方

---

# 週あたりの学習時間

| 区分 | 時間 | 内容 |
|---|---|---|
| 動画視聴 | 約20分 | 2パート×約10分 |
| 必須課題 | 約60分 | 各Session 1課題 |
| 任意課題 | 30-60分 | 追加課題（希望者向け） |
| **標準合計** | **約80分/週** | 必須のみで修了可能 |
| 最大 | 約140分/週 | 全課題取り組み時 |

**タイプB/Cは必須課題のみでOK。タイプAは任意課題で深掘り。**

---

# 各Sessionの構成

| パート | 時間 | 内容 |
|---|---|---|
| Part X-1 | 約10分 | 理論・概念説明 → **必須課題** |
| Part X-2 | 約10分 | 実践・改善パターン → 任意課題 |

```
【学習の流れ】
動画視聴 → 課題に取り組む → Webアプリで提出 → 自動評価で確認
```

**動画は常時公開。自分のペースで進められます。**

---

# 課題の評価方法

### 自動評価（Gemini API）

- 提出後、すぐに評価結果が届く
- スコア（0-100点）＋フィードバックコメント

### 評価軸（各25点）

| 軸 | 見ているポイント |
|---|---|
| 課題理解度 | 課題の意図を正しく理解しているか |
| 実践性 | 実務で使えるレベルか |
| 創意工夫 | 独自の工夫や発展があるか |
| 完成度 | 必要な要素が揃っているか |

---

# 評価結果の活用

```
┌──────────────────────────────────────────┐
│                                          │
│  スコアは「成績」ではなく「学習指針」    │
│                                          │
│  ・自分の強み/弱みを把握                 │
│  ・改善ポイントを次の課題に活かす        │
│  ・修了判定には使用しない                │
│                                          │
└──────────────────────────────────────────┘
```

**提出すれば修了できる。スコアを気にしすぎない。**

# 修了要件

| 要件 | 基準 |
|---|---|
| 動画視聴率 | **100%**（全12セッション） |
