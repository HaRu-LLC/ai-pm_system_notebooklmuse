# Part 1: GPTsとは何か

---

# GPTsとは

### カスタムAIアシスタントを作成できる機能

```
【通常のChatGPT】
毎回同じ説明が必要
↓
「あなたは営業資料作成のプロです。
 以下の条件で作成してください...」
↓
毎回コピペ → 面倒、ミスが起きる

【GPTs】
一度設定すれば繰り返し使える
↓
「営業資料作成GPTs」を呼び出すだけ
↓
すぐに作業開始 → 効率的、一貫性あり
```

---

# GPTsの活用例

### ビジネスでの活用シーン

| カテゴリ | GPTs例 | 機能 |
|---|---|---|
| 営業 | 提案書ドラフター | 顧客情報→提案書骨子 |
| 人事 | 求人票ジェネレーター | 要件→求人票作成 |
| 経理 | 経費精算ヘルパー | レシート情報→申請書 |
| CS | FAQボット | 質問→マニュアル参照回答 |
| 企画 | 企画書ライター | アイデア→企画書 |

---

# GPTsの3要素

### GPTsを構成する3つの設定

| 要素 | 役割 | 例 |
|---|---|---|
| **Instructions** | AIの振る舞いを定義 | ペルソナ、対話フロー、禁止事項 |
| **Knowledge** | 参照する知識ファイル | マニュアル、FAQ、社内規定 |
| **Actions** | 外部API連携 | データベース、カレンダー、メール |

---

# 3要素の関係

```
┌─────────────────────────────────────────────┐
│                   GPTs                       │
│                                              │
│  ┌──────────────────────────────────────┐  │
│  │ Instructions（指示）                   │  │
│  │ ・ペルソナ「営業支援のプロ」           │  │

│  │ ・禁止事項「競合情報は扱わない」       │  │
│  └──────────────────────────────────────┘  │
│           ↓ 参照                             │
│  ┌──────────────────────────────────────┐  │
│  │ Knowledge（知識ベース）               │  │
│  │ ・商品カタログ.pdf                     │  │
│  │ ・提案書テンプレート.md                │  │
│  │ ・FAQ.json                             │  │
│  └──────────────────────────────────────┘  │
│           ↓ 連携（オプション）               │
│  ┌──────────────────────────────────────┐  │
│  │ Actions（API連携）                    │  │
│  │ ・CRMデータ取得                        │  │

```

# Phase 3で扱う範囲

### 本プログラムでのフォーカス

| 要素 | 扱い | 理由 |
|---|---|---|
| **Instructions** | ✅ 重点的に学ぶ | GPTsの核心、メタプロンプトで生成 |
| **Knowledge** | ✅ 重点的に学ぶ | 精度向上の鍵、構造設計が重要 |

| 項目 | 内容 |
|---|---|
| **本プログラムの範囲外** | Actionsの実装は扱わない |
| **導入に必要な審査** | セキュリティ確認、情報システム部門承認 |
| **API開発スキル** | OAuth認証、REST API等の知識が必要 |
| **推奨アプローチ** | まずInstructions+Knowledgeで運用し、必要性を検証してからActions検討 |

# Part 2: Instructions詳解

---

# Instructionsとは

### GPTsの「振る舞い」を定義するテキスト

```
あなたは営業提案書作成のプロフェッショナルです。

【基本動作】
1. ユーザーから顧客情報をヒアリング
2. 提案書の構成案を提示
3. ユーザーの承認後、提案書を作成

【禁止事項】
- 競合他社の情報は扱わない
- 価格交渉には応じない

【出力形式】
- 提案書はMarkdown形式
- 見出し・箇条書きを活用
```

---

# Instructionsの構成要素

### 効果的なInstructionsの5要素

| 要素 | 内容 | 重要度 |
|---|---|---|
| **ペルソナ** | AIが演じる役割・専門性 | ★★★ |
| **基本動作** | 対話の流れ・処理手順 | ★★★ |
| **禁止事項** | やってはいけないこと | ★★☆ |
| **出力形式** | 回答のフォーマット | ★★☆ |
| **例外処理** | 想定外への対応 | ★☆☆ |

---

# これ、どこかで見たことが...

### Phase 2で学んだ「複合プロンプト」そのもの！

| 複合プロンプト | Instructions |
|---|---|
| ペルソナ設定 | ペルソナ定義 |
| Phase 1（要件定義） | 基本動作の前半 |
| Phase 2（実行） | 基本動作の後半 |
| ガードレール | 禁止事項 |
| 出力形式 | 出力形式ルール |

**→ Phase 2の学びがそのまま活きる！**

# Part 3: Knowledge詳解

---

# Knowledgeとは

### GPTsが参照できる知識ファイル

| 特徴 | 説明 |
|---|---|
| **ファイル形式** | PDF, TXT, Markdown, JSON, CSV等 |
| **容量制限** | GPTs 1つにつき20ファイルまで |
| **検索方式** | AIが質問に関連する部分を自動検索 |
| **用途** | マニュアル、FAQ、製品情報、テンプレート等 |

---

# Knowledgeが必要な理由

### AIの限界を補う

| AIの限界 | Knowledgeで解決 |
|---|---|
| 最新情報を知らない | 最新のドキュメントを参照 |
| 社内情報を知らない | 社内マニュアルを参照 |
| 独自ルールを知らない | 社内規定を参照 |
| 過去の事例を知らない | 過去の提案書を参照 |

---

# Knowledge設計のポイント

### 検索精度を高める工夫

| ポイント | 良い例 | 悪い例 |
|---|---|---|
| **ファイル分割** | カテゴリ別に分ける | 1つの巨大ファイル |
| **見出し構造** | 明確な階層化 | フラットな箇条書き |
| **キーワード** | 検索されやすい用語 | 曖昧な表現 |
| **形式統一** | 一貫したフォーマット | バラバラな形式 |

---

# Knowledge設計の例

### 営業提案GPTsの知識ベース

```
📁 Knowledge（知識ベース）
├── 01_商品カタログ.md
│   ├── 商品A: 概要、価格、特徴
│   ├── 商品B: 概要、価格、特徴
│   └── ...
├── 02_FAQ.md
│   ├── Q1: 導入期間は？ → A: 約2ヶ月
│   ├── Q2: サポート体制は？ → A: 24時間対応
│   └── ...
├── 03_提案書テンプレート.md
│   └── 構成例、書き方ガイド
└── 04_成功事例.md
    ├── 事例1: A社（製造業）
    ├── 事例2: B社（小売業）
    └── ...
```

---

# Knowledge運用のガバナンス

### 業務移管時に必要な管理項目

| 項目 | 内容 | 例 |
|---|---|---|
| **オーナー** | 知識ファイルの管理責任者 | 営業企画部 田中 |
| **更新頻度** | どのタイミングで更新するか | 四半期ごと、新商品追加時 |
| **版管理** | ファイル名に版番号を含める | `商品カタログ_v2.3.md` |
| **機密区分** | 公開可否、社内限定か | 社外秘、社内公開 |
| **有効期限** | いつまで有効なデータか | 2025年度版 |

> **Session 8で詳しく学ぶ**: 今は「管理が必要」という認識でOK

# Part 4: GPTs構築プロセス

---

# 従来のGPTs構築

### 手書きで一から作る

```
1. 対象業務を分析（紙やホワイトボード）
2. Instructionsを手書きで作成
3. Knowledgeファイルを手動で整理
4. GPTsビルダーで設定
5. テストして修正
6. また手書きで修正...
```

**課題**: 時間がかかる、抜け漏れが多い、品質にバラつき

---

# Phase 3のアプローチ

### すべてはメタプロンプトから始まる

```
1. 対象業務の概要を整理
   ↓ メタプロンプトに入力
2. GPTs企画書を「AIに生成してもらう」
   ↓ メタプロンプトに入力
3. Instructionsを「AIに生成してもらう」
   ↓ メタプロンプトに入力
4. Knowledge構造を「AIに提案してもらう」
   ↓
5. GPTsビルダーで設定
```

**効果**: 高速、網羅的、品質が安定

---
