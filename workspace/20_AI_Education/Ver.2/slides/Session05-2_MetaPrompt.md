# Part 1: メタプロンプトとは

# 5-1の振り返り: 変数化とテンプレート

### テンプレートの課題

| 状況 | 問題点 |
|---|---|
| 変数を埋める | どの変数に何を入れればいいか迷う |
| 新しい用途 | テンプレートを調整する必要がある |
| チーム共有 | 人によって使い方がバラバラ |

**→ テンプレートだけでは「使いこなし」に差が出る**

---

# 解決策: メタプロンプト

### 「プロンプトを生成するプロンプト」

```
【従来】
テンプレート → 変数を手動で埋める → プロンプト完成

【メタプロンプト】
メタプロンプト → AIが変数を埋めてくれる → プロンプト完成
```

**→ AIに「どんなプロンプトが必要か」を伝えるだけ！**

---

# メタプロンプトのメリット

| メリット | 効果 |
|---|---|
| **変数設定が簡単** | AIが適切な値を提案 |
| **用途適応が自動** | 目的を伝えるだけでカスタマイズ |
| **品質が安定** | 毎回同じ構造のプロンプトが生成 |
| **共有が容易** | 1つのメタプロンプトで全用途に対応 |

---

# メタプロンプトの構造

```
あなたはプロンプト設計のエキスパートです。
ユーザーの目的に応じて、最適なプロンプトを生成してください。

【生成するプロンプトの構造】
- ペルソナ設定
- Phase 1: 要件定義（改善ループ付き）
- Phase 2: 成果物生成（改善ループ付き）
- 遷移条件・終了条件
- 品質基準（DoD）
- ガードレール

ユーザーの目的: {ここにユーザーが目的を入力}
```

# Part 2: メタプロンプトの設計

---

# メタプロンプト設計の5ステップ

| ステップ | 内容 |
|---|---|
| **Step 1** | 基本構造を定義 |
| **Step 2** | 変数の自動設定ルールを作成 |
| **Step 3** | 品質基準を組み込む |
| **Step 4** | ガードレールを追加 |
| **Step 5** | テスト・改善 |

---

# Step 1: 基本構造を定義

### メタプロンプトの骨格

```
あなたはプロンプト設計のエキスパートです。

【あなたのタスク】
ユーザーが達成したい目的を聞き、
その目的に最適なプロンプトを生成してください。

【生成するプロンプトの必須要素】
1. ペルソナ（AIの役割）
2. Phase 1: 要件定義フェーズ
3. Phase 2: 実行フェーズ
4. 遷移条件と終了条件
5. 品質基準（DoD）
```

---

# Step 2: 変数の自動設定ルール

### AIに変数を推論させる

```
【変数設定ルール】
ユーザーの目的から以下を推論してください：

{役割}: 目的達成に最適な専門家を設定
{目的}: ユーザーが達成したいこと
{成果物}: 最終的なアウトプットの形式
{制約}: 業界慣行や一般的な制約を推測
{確認観点}: 目的達成に必要な確認項目を5-7個

※ 推論に自信がない項目はユーザーに確認
```

---

# Step 3: 品質基準を組み込む

### 品質基準（DoD）の自動生成

```
【品質基準の設定】
生成するプロンプトには、以下の品質基準を含めてください：

1. 目的達成度（目的を満たしているか）
2. 完成度（必要な要素が揃っているか）
3. 実用性（すぐに使える状態か）
4. 用途固有の基準（目的に応じて追加）

※ Phase 1で品質基準をユーザーと確認する
```

---

# Step 4: ガードレールを追加

### ビジネス利用の安全対策

```
【ガードレール（プロンプト本文に記載）】
⚠️ 機密情報は入力しないでください
⚠️ AIの出力は事実確認が必要です

【業務移管チェックリスト（別途確認）】
□ 機密区分: 社外秘情報を入力していないか
□ 事実確認: 数値・事実を元データで確認したか
□ 責任者: 最終承認者は明確か
□ 再利用可否: 他者が使ってよいか確認したか
```

> **方針**: 本文は2項目に絞り、責任者・再利用はチェックリストで確認

---

# Step 5: テスト・改善

### 動作確認のチェックリスト

| 確認項目 | チェック |
|---|---|
| 目的を伝えるとプロンプトが生成される | □ |
| 生成されたプロンプトが正しく動作する | □ |
| Phase 1/Phase 2の構造が含まれる | □ |
| 品質基準（DoD）が含まれる | □ |
| ガードレールが含まれる | □ |
| 不明点はユーザーに確認する | □ |

# Part 3: メタプロンプトの完成版

---

# 完成版メタプロンプト（前半）

```
あなたはプロンプト設計のエキスパートです。
ユーザーが達成したい目的を聞き、最適なプロンプトを生成してください。

【対話フロー】
1. ユーザーの目的を確認
2. 以下の変数を推論し、提案
   - {役割}: 目的達成に最適な専門家
   - {目的}: 達成したいこと
   - {成果物}: アウトプットの形式
   - {制約}: 守るべきルール
   - {確認観点}: 要件定義で確認すべき項目（5-7個）
3. ユーザーの承認を得たらプロンプトを生成
```

---

# 完成版メタプロンプト（後半）

```
【生成するプロンプトの構造】
- ペルソナ: {役割}
- Phase 1: 要件定義（まとめ→提案→質問のループ）
  - 遷移条件: 「OK」でPhase 2へ
  - 確認観点: {確認観点} + 品質基準（DoD）
- Phase 2: 成果物生成（まとめ→提案→質問のループ）
  - 終了条件: 「完了」でセッション終了
  - 制約: {制約}

【ガードレール】
⚠️ 機密情報は入力しないでください
⚠️ AIの出力は事実確認が必要です
```

---

# メタプロンプトの使い方

### 実際の対話例

**ユーザー**: 営業提案書を作りたい

**AI（メタプロンプト）**:
```
営業提案書作成ですね。以下の設定でプロンプトを生成します：

{役割} = 営業提案のプロフェッショナル
{目的} = クライアント向け営業提案書の作成
{成果物} = 10ページ程度のPowerPoint形式
{制約} = 競合比較を含める、予算感を明記
{確認観点} = 提案先、課題、予算、決裁者、納期、競合状況、差別化ポイント

この設定でよろしいですか？修正があればお知らせください。
```

---

# 生成されるプロンプト例

```
あなたは営業提案のプロフェッショナルです。

【Phase 1: 要件定義】
以下のサイクルを繰り返してください：
1. まとめ: ヒアリング内容を整理
2. 提案: 提案書の方向性を提示
3. 質問: 詳細を確認

確認観点：提案先、課題、予算、決裁者、納期、
         競合状況、差別化ポイント、品質基準

「OK」と言われたらPhase 2に進んでください。

【Phase 2: 営業提案書作成】
（続く...）
```

# チーム共有のメリット

### 1つのメタプロンプトで全員が使える

| 従来 | メタプロンプト導入後 |
|---|---|
| 人によって使い方がバラバラ | 統一された品質 |
| テンプレートの解釈に差 | AIが適切に解釈 |
| 新人への教育コスト | メタプロンプトを渡すだけ |
| 用途ごとに別テンプレート | 1つで全用途対応 |

---

# 運用のベストプラクティス

### 1. バージョン管理

```
【メタプロンプト v1.0】
- 作成日: 2024/XX/XX
- 作成者: ○○
- 対応用途: 企画書、提案書、議事録、メール

【変更履歴】
- v1.0: 初版作成
- v1.1: ガードレール追加
- v1.2: 品質基準の確認項目追加
```

---

# 運用のベストプラクティス

### 2. 用途別のプリセット

```
【プリセット集】
プリセット名を伝えるだけで、基本設定が適用されます。

■ 企画書プリセット
  {役割} = 企画書作成のベテランコンサルタント
  {確認観点} = 目的、読者、必須要素、競合状況、予算

■ 議事録プリセット
  {役割} = 議事録作成のエキスパート
  {確認観点} = 会議名、参加者、議題、決定事項、次回予定
```

---

# 運用のベストプラクティス

### 3. フィードバックの収集

```
【改善サイクル】
1. チームメンバーがメタプロンプトを使用
2. 使いにくい点・改善点をフィードバック
3. 月次でメタプロンプトを更新
4. 新バージョンをチームに展開

【フィードバック項目】
- 変数の推論精度
- 生成されるプロンプトの品質
- 不足している用途
```

# EX-10: メタプロンプト作成

### 課題内容

「プロンプトを生成するプロンプト」（メタプロンプト）を
作成し、3つの異なる目的でテストしてください。

### 提出物
1. 完成版メタプロンプト
2. 3つの目的での実行ログ
   - メタプロンプトへの入力
   - 生成されたプロンプト
   - 生成プロンプトの動作確認
3. 振り返り（メタプロンプトの効果・課題）

---

# EX-10 取り組みのポイント

### 成功のコツ

1. **5-1のテンプレートをベースにする**
   - 汎用プロンプトテンプレートを組み込む

2. **3つの目的は異なる分野を選ぶ**
   - 例: 企画書、議事録、メール
   - 同じ分野の3パターンは避ける

3. **生成されたプロンプトを必ずテストする**
   - 新しいチャットで動作確認

4. **改善点を記録する**
   - 次のバージョンに反映
