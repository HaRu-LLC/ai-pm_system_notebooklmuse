# Part 1: 出力形式のパターン

---

# なぜ出力形式が重要？

### 同じ情報でも、形式で使いやすさが変わる

| 用途 | 適した形式 |
|---|---|
| 報告書に貼り付け | 段落形式 |
| スプレッドシートで管理 | 表形式 |
| システムに取り込む | JSON形式 |
| チェックリストにする | 箇条書き |

---

# 出力形式の主なパターン

```
┌─────────────────────────────────────────┐
│                                         │
│   1. 箇条書き（リスト）                 │
│   2. 番号付きリスト                     │
│   3. 表形式（テーブル）                 │
│   4. 見出し＋段落                       │
│   5. JSON / 構造化データ                │
│   6. Markdown形式                       │
│                                         │
└─────────────────────────────────────────┘
```

---

# 1. 箇条書き（リスト）

**使いどころ**: 要点整理、アイデア出し

### プロンプト例
```
新商品の特徴を箇条書きで5つ挙げてください。
```

### 出力例
```
• 軽量設計で持ち運びやすい
• バッテリー持続時間が従来比2倍
• 防水・防塵対応
• 直感的なタッチ操作
• エコ素材を使用
```

---

# 2. 番号付きリスト

**使いどころ**: 手順説明、優先順位がある場合

### プロンプト例
```
プロジェクト開始時にやるべきことを、
優先順位の高い順に番号付きで5つ挙げてください。
```

### 出力例
```
1. プロジェクト目標の明確化
2. ステークホルダーの特定
3. スケジュール策定
4. リソース確保
5. キックオフ会議の実施
```

---

# 3. 表形式（テーブル）

**使いどころ**: 比較、一覧、スプレッドシート貼り付け

### プロンプト例
```
以下の形式の表で出力してください：
| 項目 | 説明 | 優先度 |
```

### 出力例
```
| 項目 | 説明 | 優先度 |
|---|---|---|
| 目標設定 | KPIを数値で定義 | 高 |
| 体制確認 | 担当者と役割を明確化 | 高 |
| スケジュール | マイルストーンを設定 | 中 |
```

---

# 4. 見出し＋段落

**使いどころ**: 報告書、説明文書

### プロンプト例
```
以下の構成で出力してください：
## 背景
## 課題
## 提案
## 期待効果
```

### 出力例
```
## 背景
当社の顧客対応時間は業界平均より20%長く...

## 課題
主な原因はマニュアルの分散と検索性の低さ...
```

---

# 5. JSON形式

**使いどころ**: システム連携、データ処理

### プロンプト例
```
以下のJSON形式で出力してください：
{
  "title": "タイトル",
  "summary": "要約（100文字以内）",
  "keywords": ["キーワード1", "キーワード2"]
}
```

### 出力例
```json
{
  "title": "Q4売上報告",
  "summary": "第4四半期は前年比15%増の売上を達成...",
  "keywords": ["売上", "Q4", "成長"]
}
```

---

# 6. Markdown形式

**使いどころ**: ドキュメント、Wiki、Notion

### プロンプト例
```
Markdown形式で出力してください。
見出しは##、重要語は**太字**で。
```

### 出力例
```markdown
## プロジェクト概要

本プロジェクトは**業務効率化**を目的とし、
主に以下の3点に取り組みます：

- タスク管理の自動化
- レポート作成の効率化
- ナレッジ共有の促進
```

---

# 形式指定のコツ

### 明示的に例示する

```
以下の形式で出力してください：

【タイトル】
（ここにタイトル）

【概要】
（ここに概要：3行以内）

【詳細】
- ポイント1
- ポイント2
- ポイント3
```

**→ AIは例示された形式を忠実に再現する**

# Part 2: 業務シーン別プロンプト

---

# 業務プロンプト設計の流れ

```
┌─────────────────────────────────────────┐
│                                         │
│   1. 業務シーンを特定                   │
│          ↓                              │
│   2. 4要素を設計                        │
│      （指示・文脈・制約・出力形式）     │
│          ↓                              │
│   3. 実行してテスト                     │
│          ↓                              │
│   4. 微調整して再利用可能に             │
│                                         │
└─────────────────────────────────────────┘
```

---

# 業務シーン①: メール作成

### Before
```
お礼メールを書いて
```

### After
```
【指示】取引先への商談お礼メールを作成してください。

【文脈】私は営業部の山田です。
昨日、株式会社ABC様と新規案件の商談を行いました。
先方の担当は鈴木部長です。

【制約】
- 200文字程度
- 敬語で丁寧に
- 次回のアポイント日程確認を含める

【出力形式】
件名と本文を分けて出力
```

---

# メール作成プロンプトの出力例

```
【件名】
昨日のご面談のお礼（株式会社〇〇・山田）

【本文】
株式会社ABC
鈴木部長

お世話になっております。
株式会社〇〇の山田でございます。

昨日はお忙しい中、貴重なお時間をいただき
誠にありがとうございました。

ご提案内容についてご検討いただけますと幸いです。
次回のお打ち合わせ日程について、
ご都合の良い日時をお知らせいただけますでしょうか。

引き続きよろしくお願いいたします。
```

---

# 業務シーン②: 議事録作成

### プロンプト例
```
【指示】以下の会議メモから議事録を作成してください。

【文脈】週次の営業会議（参加者5名）の議事録です。
部門全体で共有するため、結論とアクションを明確に。

【制約】
- 箇条書きで簡潔に
- 専門用語はそのまま使用OK
- 決定事項とTODOは必ず明記

【出力形式】
## 日時・参加者
## 議題
## 決定事項
## アクションアイテム（担当・期限付き）

【会議メモ】
（ここに生のメモを貼り付け）
```

---

# 業務シーン③: レポート要約

### プロンプト例
```
【指示】以下のレポートを経営層向けに要約してください。

【文脈】四半期の業績報告書です。
取締役会で報告するため、重要指標に絞って要約します。

【制約】
- 300文字以内
- 専門用語は避け、平易な表現で
- 数値は前年比・前期比を併記
- ネガティブな内容も隠さず記載

【出力形式】
### サマリー（3行）
### 主要指標（表形式）
### 次四半期の見通し

【レポート本文】
（ここにレポートを貼り付け）
```

---

# 業務プロンプト設計の5原則

| # | 原則 | 説明 |
|---|---|---|
| 1 | 具体的に | 「良い」ではなく「300文字以内」 |
| 2 | 例示する | 出力形式は例を書く |
| 3 | 役割を与える | ペルソナで専門性を引き出す |
| 4 | 読み手を意識 | ターゲットで表現を調整 |
| 5 | 再利用を考える | テンプレート化しやすく |

---

# 再利用可能なテンプレート化

### 変数部分を〇〇で置き換える

```
【指示】〇〇（相手）への〇〇（目的）メールを作成してください。

【文脈】私は〇〇部の〇〇です。
〇〇という状況で、〇〇のためにメールを送ります。

【制約】
- 〇〇文字程度
- 〇〇なトーンで
- 〇〇を必ず含める

【出力形式】
件名と本文を分けて出力
```

# EX-04: 業務プロンプト3本セット【任意】

**課題内容**:
実際の業務で使えるプロンプトを3種類作成する

**成果物**:
以下の3種類のプロンプトと実行結果

2. 議事録作成プロンプト
3. レポート要約プロンプト（または自由選択）

**評価基準**:

---

# EX-04 の取り組み方

### ステップ1: 業務シーンを具体化
- 実際に自分がやっている業務を選ぶ
- 「これをAIに任せたい」を考える

### ステップ2: 4要素を設計
- テンプレートを活用して埋める
- 出力形式は例示を含める

### ステップ3: 実行してブラッシュアップ
- 1回で完璧は難しい
- 「ここをもっと」を追記して再実行

---

# 課題のポイント

### やってみよう
1. 自分の業務で本当に使うものを選ぶ
2. 「明日から使える」レベルを目指す
3. 同僚に見せても恥ずかしくない品質

### 困ったら
- **テンプレートを使う**（ポータルからダウンロード可能）
- 動画内の例をアレンジしてみる
- 完璧を目指さず、まず「動く」ものを作る
