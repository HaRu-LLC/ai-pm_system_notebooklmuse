# Part 1: 2段階プロンプトとは

---

# 1回完結型の問題点（再確認）

### すべてを最初に指定する難しさ

```
【指示】マーケティング企画書を作成してください。
【文脈】〇〇で、△△な状況で...
【制約】〇〇文字で、△△を含めて...
【出力形式】〇〇の形式で...
```

**問題**:
- 最初から全ての要件を把握できない
- 書いてみないとわからないことがある
- 「これじゃない」の手戻りが発生

---

# 2段階プロンプトの発想

```
┌─────────────────────────────────────────┐
│                                         │
│  【Phase 1】要件定義フェーズ            │
│   - 何を作るか？                        │
│   - どんな条件か？                      │
│   - 何を重視するか？                    │
│   → 要件を明確化してから...            │
│                                         │
│  【Phase 2】実行フェーズ                │
│   - 確定した要件に基づいて作成          │
│   - ブレなく一貫した出力                │
│                                         │
└─────────────────────────────────────────┘
```

---

# なぜ2段階に分けるのか

| 観点 | 1回完結型 | 2段階型 |
|---|---|---|
| 要件の明確さ | 曖昧なまま進む | 確認してから進む |
| 手戻りリスク | 高い | 低い |
| 出力の一貫性 | バラつきやすい | 安定しやすい |
| ユーザー負担 | 最初に全て決める | 対話で決める |
| 所要時間 | 短いが手戻り発生 | 初回は長いが効率的 |

---

# 2段階プロンプトの構造

```
【Phase 1: 要件定義】

あなたは〇〇です。
私が〇〇を作成できるよう、まず要件を整理してください。

以下について質問し、私の回答をもとに要件をまとめてください：
- 目的
- 対象読者
- 含めるべき要素
- 制約条件
- 重視するポイント

私が「要件OK」と言ったら、Phase 2に進んでください。
```

---

# 2段階プロンプトの構造（続き）

```
【Phase 2: 実行】

Phase 1で確定した要件に基づいて、〇〇を作成してください。

### 要件（Phase 1で確定したもの）
（ここにまとめが入る）

### 出力形式
〇〇

私が「完了」と言うまで、修正に応じてください。
```

# Part 2: 要件定義フェーズの設計

---

# 要件定義で押さえる5つの観点

| # | 観点 | 質問例 |
|---|---|---|
| 1 | 目的 | 何のために作るのか？ |
| 2 | 対象 | 誰が読む/使うのか？ |
| 3 | 内容 | 何を含めるべきか？ |
| 4 | 制約 | 長さ、形式、禁止事項は？ |
| 5 | 優先 | 何を最も重視するか？ |

---

# 品質基準（Definition of Done）を追加する

### 要件定義に「完成の定義」を含める

| 観点 | 品質基準の例 |
|---|---|
| **必須要素** | 〇〇と△△が含まれていること |
| **受け手基準** | 読者が5分で理解できること |
| **確認観点** | 上司に見せる前に自分でチェックする3点 |

### プロンプトへの追加例
```
### 品質基準
以下を満たしていることを確認してから完成とします：
- 〇〇が含まれている
- △△のトーンで書かれている
- □□の形式になっている
```

---

# 要件定義フェーズのプロンプト例

```
あなたは経験豊富なビジネスコンサルタントです。

私が企画書を作成できるよう、まず要件を整理してください。

### 確認事項
以下について、1つずつ質問してください：
1. 企画の目的（何を達成したいか）
2. 読者（誰がこの企画書を読むか）
3. 必須要素（必ず含めるべき内容）
4. 制約（ページ数、形式、締切など）
5. 重視するポイント（インパクト重視 or 実現性重視など）

### 次のステップ
全ての確認が終わったら、要件を箇条書きでまとめてください。
私が「要件OK」と言ったら、Phase 2（実際の作成）に進みます。
```

---

# 要件定義の対話例

```
AI: 「企画書の作成を支援します。
      まず、この企画の目的を教えてください。
      何を達成したいですか？」

ユーザー: 「社内のDX推進予算を獲得したい」

AI: 「DX推進予算の獲得が目的ですね。
      次に、誰がこの企画書を読みますか？
      承認者は誰でしょうか？」

ユーザー: 「経営会議で役員が読む」

AI: 「役員向けの経営会議資料ですね。
      必ず含めるべき内容はありますか？
      例：ROI試算、競合比較、リスク分析など」

（続く...）
```

---

# 要件まとめの例

```
### 確定した要件

■ 目的
- 社内DX推進プロジェクトの予算獲得（1,500万円）

■ 対象読者
- 経営会議の役員（意思決定者）
- 技術的な詳細より、ビジネスインパクト重視

■ 必須要素
- 現状課題と解決策
- ROI試算（3年間）
- 競合他社の動向
- リスクと対策

■ 制約
- A4で10ページ以内
- 図表を多用（文字だけのページを避ける）

■ 重視するポイント
- 投資対効果の説得力
```

---

# 「要件OK」で次のフェーズへ

### ユーザーの確認アクション

```
ユーザー: 「要件OK、作成に進んで」
```

または

```
ユーザー: 「リスク分析は不要。それ以外はOK」

AI: 「承知しました。リスク分析を除外して、
      以下の要件で作成を開始します...」
```

**ポイント**: 明確な承認アクションでフェーズ遷移

# Part 3: 実行フェーズの設計

---

# 実行フェーズの役割

```
┌─────────────────────────────────────────┐
│                                         │
│  【入力】確定した要件                   │
│     ↓                                   │
│  【処理】要件に忠実に作成               │
│     ↓                                   │
│  【出力】成果物（ドラフト）             │
│     ↓                                   │
│  【修正】フィードバックに応じて改善     │
│                                         │
└─────────────────────────────────────────┘
```

---

# 実行フェーズのプロンプト例

```
【Phase 2: 作成】

以下の確定要件に基づいて、企画書を作成してください。

### 確定要件
- 目的: DX推進予算1,500万円の獲得
- 読者: 経営会議の役員
- 必須要素: 現状課題、ROI試算、競合動向
- 制約: A4 10ページ以内、図表多用
- 重視: 投資対効果の説得力

### 出力形式
ページごとにセクションを分けて出力してください。
まず目次案を提示し、承認後に各ページを作成します。

### 修正対応
私が「完了」と言うまで、修正に応じてください。
```

---

# 実行フェーズのポイント

### 1. 要件を明示的に再掲する

```
### 確定要件
（Phase 1で決めた内容を箇条書きで再掲）
```

**理由**: AIが「忘れない」ように、参照可能な形で提示

---

# 実行フェーズのポイント（続き）

### 2. 段階的に出力する

```
まず目次案を提示してください。
承認後、各セクションを順番に作成します。
```

**理由**:
- 大きな成果物を一度に出力すると品質が落ちる
- 途中で方向修正しやすい
- トークン制限にも対応

---

# 実行フェーズのポイント（続き）

### 3. 修正ループを組み込む

```
私が「完了」と言うまで、修正に応じてください。
修正点を伝えたら、該当箇所のみ更新してください。
```

**理由**:
- 一度で完璧にならないことを前提にする
- 部分修正で効率化
- Session 3-1の改善ループを適用

---

# 2段階プロンプト全体像

```
【Phase 1】
あなたは〇〇です。まず要件を整理します。
（5つの観点で質問）
→ 要件まとめを提示
→ 「要件OK」で Phase 2へ

【Phase 2】
確定要件に基づいて作成します。
（要件を再掲）
→ 目次案提示
→ 承認後、各セクション作成
→ 「完了」まで修正対応
```

# 業務活用のガードレール

### AI出力を業務で使う前の確認事項

| 確認項目 | チェック内容 |
|---|---|
| **機密区分** | 社外秘情報をAIに入力していないか？ |
| **事実確認** | AIが出力した数値・事実は正確か？ |
| **責任者** | 最終的な承認・責任は誰が持つか？ |
| **再利用** | この成果物は他者が再利用してよいか？ |

### プロンプトへの組み込み例
```
### 注意事項
- この資料は社内検討用です
- 数値は必ず元データで確認してください
- 最終版は〇〇の承認を得てください
```
