# 4要素の復習

```
┌─────────────────────────────────────────┐
│                                         │
│         「し・ぶ・せ・だ」              │
│                                         │
│   し = 指示（何をしてほしい？）         │
│   ぶ = 文脈（どんな状況？）             │
│   せ = 制約（どんな条件？）             │
│   だ = 出だし形式（どう返す？）         │
│                                         │
└─────────────────────────────────────────┘
```

---

# Before/After比較①: 自己紹介

### Before（曖昧なプロンプト）
```
自己紹介書いて
```

### 問題点
- ❌ 誰の自己紹介？
- ❌ どんな場面で使う？
- ❌ どのくらいの長さ？
- ❌ どんなトーン？

**→ AIは推測するしかない**

---

# Before/After比較①: 改善後

### After（4要素を含むプロンプト）
```
【指示】自己紹介文を作成してください。

【文脈】転職活動中のIT企業向け面接で使用します。
私は営業職5年目、主にBtoB法人営業を担当してきました。

【制約】1分で話せる長さ（約300文字）、
熱意が伝わるトーンで。

【出力形式】話し言葉で、段落分けして出力してください。
```

# Before/After比較②: メール作成

### Before
```
メールを書いて
```

### After
```
【指示】取引先へのお礼メールを作成してください。
【文脈】先日の商談で契約が決まりました。
【制約】丁寧な敬語、簡潔に（200文字程度）
【出力形式】件名と本文を分けて
```

# Before/After比較③: 議事録作成

### Before
```
議事録作って
```

### After
```
【指示】以下の会議メモから議事録を作成してください。
【文脈】営業部の週次定例会議（参加者5名）
【制約】正式な議事録形式、箇条書きで簡潔に
【出力形式】
- 日時・参加者
- 議題
- 決定事項
- アクションアイテム（担当・期限付き）

【会議メモ】
（ここに生のメモを貼り付け）
```

---

# プロンプト改善の3ステップ

### 1. まず書いてみる
最初から完璧を目指さない

### 2. 結果を見て改善する
「もっと〇〇してほしかった」を言語化

### 3. 要素を追加していく
指示 → 文脈 → 制約 → 出力形式の順で

---

# よくある改善パターン

| 問題 | 改善方法 |
|---|---|
| 長すぎる | 「〇〇文字以内で」と制約追加 |
| 専門的すぎる | 「初心者にもわかるように」と指定 |
| 的外れ | 文脈情報を追加 |
| 形式が違う | 出力形式を明示 |
| 抽象的 | 具体例を求める |

---

# 改善の対話例

```
あなた: 「もっと具体的にしてください」

あなた: 「箇条書きではなく、文章で書き直してください」

あなた: 「専門用語を使わずに、
         中学生でもわかるように説明してください」

あなた: 「この部分をもう少し詳しく説明してください」
```

**AIとの対話で結果を磨いていく**

---

# 1-2 まとめ

### キーポイント

> **良いプロンプト = 良い応答**

1. **明確に伝える**（AIは読心術ができない）
2. **4要素を意識する**（指示・文脈・制約・出力形式）
3. **改善を繰り返す**（1回で完璧を目指さない）

# EX-02: プロンプト改善比較【任意】

**課題内容**:
同じ目的のプロンプトをBefore/Afterで比較

**成果物**:
- Before プロンプトと応答
- After プロンプトと応答

**評価基準**:
改善ポイントを論理的に説明できる

---

# EX-02のポイント

### やってみよう
1. まず「悪いプロンプト」でAIに聞いてみる
2. 結果の問題点を特定する
3. 4要素を使って改善したプロンプトを作成
4. 改善後の結果を比較

### 比較のコツ
- **何が変わったか**を具体的に記述
- **なぜ改善されたか**の理由を添える
- **テンプレートを使う**（ポータルからダウンロード可能）
