# AI適用案の具体化

### 「AIで自動化」を具体的に分解する

| 抽象的な案 | 具体的な案 | GPTsでの実現方法 |
|---|---|---|
| 自動生成 | テンプレート＋変数で生成 | Instructions + 出力形式 |
| 情報検索 | Knowledge参照で回答 | Knowledge + 検索指示 |
| チェック | ルールに基づく確認 | チェックリスト + 判定ロジック |
| 要約 | 構造化して要点抽出 | 要約プロンプト |
| 分類 | カテゴリ判定 | 分類ルール + 例示 |

# 効果の見積もり方

### 定量化のコツ

```
【時間削減の計算例】

現状: 見積書作成 30分/件 × 週20件 = 週10時間

GPTs化後（見込み）:
- AI生成: 5分/件
- 人のレビュー: 5分/件
- 合計: 10分/件 × 週20件 = 週3.3時間

削減効果: 10時間 - 3.3時間 = 週6.7時間（67%削減）

【注意】
- 最初は保守的に見積もる（50%くらいの効果で計算）
- 導入初期は習熟期間がある（最初の1-2週間は逆に時間かかる）
```

---

# 実現性の評価

### GPTs単体で実現できるかの判断

| レベル | 説明 | 例 |
|---|---|---|
| **高** | GPTsのInstructions + Knowledgeで完結 | FAQ対応、文書生成 |
| **中** | Actions（API連携）が必要 | 在庫確認、予約システム |
| **低** | 専用システム開発が必要 | 基幹システム連携 |

> **Phase 4の範囲**: 高〜中（GPTs + 簡易なActions）まで

# 優先順位マトリクス

### 効果×実現性で4象限に分類

```
        高 ┌─────────────────┬─────────────────┐
           │                 │                 │
     効    │   △ 要検討     │  ◎ 最優先      │
     果    │                 │                 │
           │  効果は高いが   │  今すぐ取り組む │
           │  実現に課題あり │                 │
           ├─────────────────┼─────────────────┤
           │                 │                 │
           │   × 見送り     │  ○ 余裕あれば  │
           │                 │                 │
           │  優先度低       │  簡単だが効果薄 │
           │                 │                 │
        低 └─────────────────┴─────────────────┘
                  低              高
                       実現性
```

---

# 優先順位の判断根拠を明記する

### なぜその優先順位なのかを言語化

```markdown
【優先順位の判断根拠】

◎ 見積書作成（優先度: 最優先）
- 効果: 高（週10時間→3時間、67%削減）
- 実現性: 高（テンプレート＋Knowledge化で完結）
- 根拠: 頻度が高く、GPTsで完結可能。すぐに効果が出る

○ 価格表検索（優先度: 余裕あれば）
- 効果: 中（週3時間→1.5時間、50%削減）
- 実現性: 高（CSVをKnowledge化するだけ）
- 根拠: 効果は中程度だが、実現が容易

△ 上長承認フロー（優先度: 要検討）
- 効果: 高（待ち時間1日→即時）
- 実現性: 低（ワークフローシステム導入が必要）
- 根拠: 効果は高いが、GPTs単体では実現困難
```

# 1つに絞る

### Phase 4では1つのGPTsを完成させる

```
【絞り込みの考え方】

AI適用候補リスト
├── ◎ 見積書作成 ← これを選ぶ
├── ○ 価格表検索
├── ○ ヒアリングチェック

【選定基準】
1. 優先度が最も高いもの（◎）
2. 自分が頻繁に行う作業
3. 2週間で完成できる規模
4. 成果がわかりやすいもの

→ Session 12-13で「見積書作成GPTs」を構築
```

# よくある悩みと対処法

### 優先順位付けでつまずくポイント

| 悩み | 対処法 |
|---|---|
| 効果の見積もりが難しい | まず1週間だけ時間を計測してみる |
| 全部優先度◎になる | 強制的に1つだけ◎にする |
| 実現性が判断できない | 「GPTsだけで完結するか」で判断 |
| どれも選べない | 一番よくやる作業を選ぶ |
| 上司の承認が必要 | 「効果×時間」で説明資料を作る |

# GPTs化方針とは

### 選定した作業をどうGPTsにするかの計画

```markdown
【GPTs化方針書】

## 1. 対象作業
- 作業名: 見積書作成
- 現状の課題: 手作業で30分/件
- 目標: 10分/件（67%削減）

## 2. GPTs構成
- Instructions: 見積書ドラフト生成
- Knowledge: 価格表.csv、過去事例.md

## 3. ユースケース
- 入力: 顧客名、製品、数量
- 出力: 見積書ドラフト（Markdown）

## 4. 成功基準
- 作成時間: 30分→10分
- 精度: 価格間違いゼロ
```
