# Part 1: Phase 1の振り返り

---

# Phase 1で学んだ2つのパターン

### Session 3で習得した対話パターン

| パターン | 特徴 | 使いどころ |
|---|---|---|
| **改善ループ** | まとめ→提案→質問を繰り返す | 探索的なタスク、アイデア出し |
| **要件定義→実行** | Phase 1で要件確定、Phase 2で作成 | 成果物が明確なタスク |

---

# 2つのパターンの限界

### 改善ループの課題
- 何を作るか曖昧なまま改善が進む
- ゴールが見えにくい

### 要件定義→実行の課題
- 要件定義が1回で終わってしまう
- 実行後の微調整がしにくい

**→ 両方の良いところを組み合わせたい！**

<!-- _class: lead -->
# Part 2: 複合プロンプトとは

---

# 複合プロンプトの構造

```
【Phase 1: 要件定義フェーズ】
  ↓ 改善ループ（まとめ→提案→質問を繰り返す）
  ↓ ユーザー「OK、これで進めて」

【Phase 2: 実行フェーズ】
  ↓ 成果物を生成
  ↓ 改善ループ（まとめ→提案→質問を繰り返す）
  ↓ ユーザー「完了」まで続ける
```

---

# 複合プロンプトのメリット

| 観点 | 従来のパターン | 複合プロンプト |
|---|---|---|
| 要件定義 | 1回で確定 | 対話で深掘り |
| 実行 | 一発勝負 | 改善を繰り返す |
| フェーズ遷移 | 手動で切り替え | ユーザー確認で自動遷移 |
| 終了条件 | 曖昧 | 明確な終了宣言 |

---

# フェーズ遷移のポイント

### ユーザーによるフェーズ遷移

```
【Phase 1】
AI: 「以下の要件でよろしいですか？」
ユーザー: 「OK、これで進めて」← フェーズ遷移トリガー

【Phase 2】
AI: 成果物を生成
AI: 「改善点はありますか？」
ユーザー: 「完了」← 終了トリガー
```

---

# 複合プロンプトの全体像

```
あなたは〇〇の専門家です。

【Phase 1: 要件定義】
以下のサイクルを繰り返してください：
1. まとめ: 私が伝えた内容を整理
2. 提案: 要件案を提示
3. 質問: 不明点を確認

私が「OK」と言ったらPhase 2に進んでください。

【Phase 2: 実行】
確定要件に基づいて成果物を作成してください。
以下のサイクルを繰り返してください：
1. まとめ: 現在の成果物の状態を説明
2. 提案: 改善案を提示
3. 質問: 修正の方向性を確認

私が「完了」と言うまで続けてください。
```

# Part 3: 複合プロンプトの設計

---

# 設計ステップ

| ステップ | 内容 | ポイント |
|---|---|---|
| 1 | ペルソナ設定 | 専門性と役割を明確に |
| 2 | Phase 1設計 | 要件定義の観点を列挙 |
| 3 | 遷移条件設定 | ユーザーの確認ワード |
| 4 | Phase 2設計 | 成果物生成と改善ループ |
| 5 | 終了条件設定 | 完了の明確化 |

---

# ステップ1: ペルソナ設定

### 良いペルソナの例

```
あなたは企画書作成のベテランコンサルタントです。
クライアントの要望を丁寧に聞き取り、
説得力のある企画書を作成することが得意です。
```

### ポイント
- 専門領域を明確に
- 作業スタイルを示す
- 成果物のイメージを共有

---

# ステップ2: Phase 1設計

### 要件定義の5観点（復習）

| 観点 | 確認内容 |
|---|---|
| 目的 | 何を達成したいか？ |
| 対象 | 誰が読む/使うか？ |
| 内容 | 何を含めるべきか？ |
| 制約 | 長さ、形式、禁止事項は？ |
| 優先 | 何を最も重視するか？ |

**→ これらを改善ループで深掘りする**

---

# ステップ3: 遷移条件設定

### フェーズ遷移のトリガー

```
【OK系】Phase 1 → Phase 2への移行
- 「OK」
- 「これで進めて」
- 「要件確定」
- 「作成開始」

【NG系】Phase 1を継続
- 「〇〇を追加して」
- 「〇〇は不要」
- 「もう少し詳しく」
```

---

# ステップ4: Phase 2設計

### 実行フェーズの改善ループ

```
【Phase 2の動作】
1. 確定要件に基づいて成果物を作成
2. 現在の成果物を提示
3. 改善点の有無を確認
4. ユーザーフィードバックに基づいて修正
5. 2-4を繰り返す
```

---

# ステップ5: 終了条件設定

### 完了トリガーの設定

```
【完了系】セッション終了
- 「完了」
- 「これで終了」
- 「OKです」
- 「ありがとう、終わり」

【継続系】修正を続ける
- 「〇〇を直して」
- 「もう少し〇〇に」
- 「追加で〇〇」
```

# 例: 企画書作成の複合プロンプト

```
あなたは企画書作成のベテランコンサルタントです。

【Phase 1: 要件定義】
以下のサイクルを繰り返してください：
1. まとめ: 私が伝えた企画の要点を整理
2. 提案: 企画書の構成案を提示
3. 質問: 目的・読者・必須要素・制約・重視点について確認

私が「OK」「進めて」と言ったらPhase 2に進んでください。

【Phase 2: 企画書作成】
確定した要件に基づいて企画書を作成してください。
作成後、以下のサイクルを繰り返してください：
1. まとめ: 現在の企画書の概要を説明
2. 提案: 改善できる点を提示
3. 質問: 修正の方向性を確認

私が「完了」と言うまで続けてください。
```

---

# 複合プロンプトの対話例（Phase 1）

```
ユーザー: 「新規事業の企画書を作りたい」

AI: 【まとめ】新規事業の企画書作成ですね。
    【提案】以下の構成はいかがでしょうか：
    - 事業概要
    - 市場分析
    - 収益モデル
    【質問】対象読者は誰ですか？経営層、投資家、部門責任者？

ユーザー: 「経営層向け。AI活用の新規サービス」

AI: 【まとめ】経営層向け、AI活用新規サービスの企画書
    【提案】経営層に響くポイント：
    - ROI試算を前面に
    - 競合優位性を明確に
    【質問】予算感と期待する投資回収期間は？

ユーザー: 「OK、これで進めて」← Phase 2へ遷移
```

---

# 複合プロンプトの対話例（Phase 2）

```
AI: 【Phase 2開始】
    確定要件に基づいて企画書を作成しました。

    ===== AI活用新規サービス企画書 =====
    1. エグゼクティブサマリー
    ...（企画書本文）...

    【まとめ】経営層向けにROI重視の構成としました。
    【提案】グラフを追加すると説得力が増します。
    【質問】他に追加したい情報はありますか？

ユーザー: 「競合比較の表を追加して」

AI: 【修正版を作成】
    ...競合比較表を追加...

ユーザー: 「完了」← セッション終了
```

---

# 品質基準（DoD）の組み込み

### Phase 1で品質基準も確認する

```
【Phase 1: 要件定義】
以下についても確認してください：
6. 品質基準: 完成とみなす条件は何か？
   - 必須要素
   - 受け手基準
   - 確認観点

→ Phase 2で成果物が品質基準を満たしているか確認
```

---

# 業務活用のガードレール（復習）

### 複合プロンプトでも忘れずに

| 確認項目 | チェック内容 |
|---|---|
| **機密区分** | 社外秘情報をAIに入力していないか？ |
| **事実確認** | AIが出力した数値・事実は正確か？ |
| **責任者** | 最終的な承認・責任は誰が持つか？ |
| **再利用** | この成果物は他者が再利用してよいか？ |

# Part 5: まとめと課題

---

# 4-1 まとめ

### 複合プロンプトとは
- 改善ループ + 要件定義→実行の組み合わせ
- Phase 1（要件定義）とPhase 2（実行）の2段階構造
- 両フェーズで改善ループを適用

### 設計の5ステップ
1. ペルソナ設定
2. Phase 1設計（要件定義の観点）
3. 遷移条件設定（「OK」で移行）
4. Phase 2設計（成果物生成＋改善ループ）
5. 終了条件設定（「完了」で終了）

---

# 必須課題 EX-07

---

# EX-07: 複合プロンプト設計

### 課題内容

複合プロンプト（Phase 1: 要件定義 + Phase 2: 実行）を設計し、
実際に使用してください。

### 提出物
1. 複合プロンプト（Phase 1 + Phase 2）
2. Phase 1の対話ログ（要件定義フェーズ）
3. Phase 2の対話ログ（実行＋改善フェーズ）
4. 最終成果物
5. 振り返り（Phase 1とPhase 2それぞれの改善効果）

---

# EX-07 取り組みのポイント

### 成功のコツ

1. **明確な遷移条件を設定する**
   - 「OK」「進めて」などのトリガーワードを決める

2. **両フェーズで改善ループを回す**
   - Phase 1: 要件を深掘り
   - Phase 2: 成果物を磨き上げ

3. **品質基準を忘れずに**
   - Phase 1でDoDを確認
   - Phase 2でDoD達成を確認
