# Part 1: 発表の構成

---

# 発表構成（20分）

### 推奨構成

| パート | 内容 |
|---|---|
| **1. 業務と課題** | 対象業務、ボトルネック、なぜGPTsか |
| **2. GPTs設計** | 企画、Instructions、知識ベース |
| **3. 実運用の結果** | 使用ログ、フィードバック、改善 |
| **4. 振り返りと今後** | 学び、今後の展望 |
| **5. 質疑応答** | 講師・受講者からの質問 |

---

# パート1: 業務と課題

### 伝えるべきこと

```
【必須要素】

□ 対象業務の説明（誰が、何を、どのくらいの頻度で）
□ 課題・ボトルネックの特定（時間、品質、属人化など）
□ なぜGPTsで解決できると考えたか
□ 期待した効果（数値目標があれば）

【例】
「営業部門で週次レポートを作成しています。
 毎週3時間かかり、フォーマットにばらつきがありました。
 定型部分をGPTsで自動化し、1時間に短縮することを目指しました。」
```

---

# パート2: GPTs設計

### 伝えるべきこと

```
【必須要素】

□ GPTsの名前と目的（1文で）
□ 主要機能（3つ程度）
□ Instructionsの設計ポイント（工夫した点）
□ 知識ベースの構成（何を、どの形式で）

【デモのポイント】
・1つの具体的なシナリオで実演
・入力→出力の流れを見せる
```

---

# パート3: 実運用の結果

### 伝えるべきこと

```
【必須要素】

□ 使用回数・使用者（自分だけ or 同僚も）
□ 代表的な使用ログ（成功例・改善例）
□ フィードバックの内容（良かった点・改善点）
□ 実施した改善とその効果
□ 定量的な成果（時間削減、品質向上など）

【数字で語る】
・「便利でした」ではなく「3時間→1時間に短縮」
・「改善しました」ではなく「2回目から精度90%に向上」
```

---

# パート4: 振り返りと今後

### 伝えるべきこと

```
【必須要素】

□ 技術面の学び（メタプロンプト、GPTs設計など）
□ プロセス面の学び（業務分析、改善サイクルなど）
□ 今後の展開（継続運用、他業務への展開など）
□ 本プログラム全体を通じた気づき

【KPT形式で整理】
・Keep: 続けたいこと
・Problem: 課題だったこと
・Try: 次に試したいこと
```

# Part 2: 発表資料の作成

---

# スライド構成の推奨

### 10-12枚程度

| スライド# | 内容 |
|:---:|---|
| 1 | タイトル（GPTs名、発表者名） |
| 2 | 業務概要と課題 |
| 3 | GPTsの目的と期待効果 |
| 4 | GPTs設計概要（機能一覧） |
| 5 | Instructions設計のポイント |
| 6 | 知識ベース構成 |
| 7 | デモ（実演 or スクリーンショット） |
| 8 | 実運用の結果（使用ログ） |
| 9 | 改善の実施と効果 |
| 10 | 振り返り（KPT） |
| 11 | 今後の展望 |
| 12 | まとめ |

---

# スライド作成のコツ

### 見やすく、伝わりやすく

```
【DO（やるべき）】

✅ 1スライド1メッセージ
✅ 箇条書きは3-5項目まで
✅ 図・スクリーンショットを活用
✅ 数字を入れる（具体性）
✅ フォントは大きく（24pt以上）

【DON'T（避けるべき）】

❌ 文章をそのまま貼り付け
❌ 小さい文字でびっしり
❌ 読み上げるだけの発表
❌ デモが長すぎる
```

---

# スクリーンショットの撮り方

### 証跡として重要

```
【撮るべきスクリーンショット】

1. GPTs設定画面
   - Instructions全体が見える状態
   - 知識ベースのファイル一覧

2. 対話ログ
   - 成功した対話例（入力→出力）
   - 改善前後の比較

3. 使用ログ（スプレッドシート等）
   - 使用回数、評価が見える状態

【撮り方のコツ】
・個人情報、機密情報はマスキング
・ウィンドウサイズを統一
・説明用の注釈を追加（矢印、囲み）
```

# 最終課題 提出物一覧

### 発表前に確認

| # | 提出物 | 形式 | 必須/任意 |
|:---:|---|---|:---:|
| 1 | 発表スライド | PDF/PPTX | **必須** |
| 2 | GPTs（実働するもの） | GPTs URL | **必須** |
| 3 | 使用ログ | Excel/スプレッドシート | **必須** |
| 4 | GPTs仕様書 | Markdown/Word | **必須** |
| 5 | 振り返りシート（KPT） | Markdown/Word | **必須** |
| 6 | フィードバック収集結果 | 任意形式 | 任意 |

---

# 提出方法と期限

### 発表前日までに提出

```
【提出方法】

1. 指定の共有フォルダにアップロード
   フォルダ名: 「最終課題_[氏名]」

2. ファイル命名規則
   - 発表スライド: [氏名]_発表スライド.pdf
   - 使用ログ: [氏名]_使用ログ.xlsx
   - 仕様書: [氏名]_GPTs仕様書.md
   - 振り返り: [氏名]_振り返りKPT.md

【提出期限】

発表前日 23:59まで
※ 遅延の場合は事前に講師へ連絡
```

---

# 提出前チェックリスト

### 漏れがないか確認

```
【チェックリスト】

□ 発表スライド
  □ 10-12枚程度にまとまっている
  □ スクリーンショットが含まれている
  □ 個人情報・機密情報がマスキングされている

□ GPTs
  □ 正常に動作する
  □ URLが共有可能な状態

□ 使用ログ
  □ 3回以上の使用記録がある
  □ 評価（○△×）が記入されている

□ 仕様書
  □ 7セクション構成が満たされている
  □ Instructions全文が含まれている

□ 振り返り
  □ KPT形式で整理されている
  □ 具体的な学びが記載されている
```
