# 評価の観点

### 5つの評価軸

| 観点 | 確認事項 | 配点 |
|---|---|---|
| **完全性** | 5構成要素がすべて含まれているか | 20点 |
| **具体性** | 抽象的でなく具体的な記述か | 20点 |
| **一貫性** | 企画書の内容と整合しているか | 20点 |
| **実用性** | そのままGPTsに設定できるか | 20点 |
| **安全性** | 禁止事項・例外処理が適切か | 20点 |

---

# 評価例: ペルソナ定義

### Good Example

```markdown
# ペルソナ
あなたは株式会社〇〇の営業支援AI「提案書ドラフターPro」です。
10年以上の法人営業経験を持つベテランマネージャーとして、
提案書作成をサポートします。

トーン: フレンドリーだがプロフェッショナル
一人称: 「私」
敬語: 使用する
```

### Bad Example

```markdown
あなたは提案書を作るAIです。
→ 専門性・トーンが不明確
```

---

# 評価例: 基本動作指示

### Good Example

```markdown
# 基本動作
1. ユーザーから顧客情報（業種・規模・課題）を受け取る
2. 顧客のニーズを分析し、3つの提案パターンを検討する
3. ユーザーが選択したパターンでドラフトを生成する
4. フィードバックを受けて最大3回まで修正する
```

### Bad Example

```markdown
提案書を作成します。
→ 具体的な動作が不明
```

---

# 評価例: 対話フロー

### Good Example

```markdown
# 対話フロー

## Phase 1: 要件ヒアリング
以下のサイクルを繰り返す：
1. まとめ: ユーザーの発言を箇条書きで整理
2. 提案: 提案書の方向性を2-3案提示
3. 質問: 不明点を1つだけ確認

ユーザーが「OK」「進めて」と言ったらPhase 2へ

## Phase 2: ドラフト作成
...
```

### Bad Example

```markdown
質問してから作成します。
→ 具体的なフローが不明
```

---

# 評価例: 禁止事項

### Good Example

```markdown
# 禁止事項
- 競合他社（A社、B社、C社）の製品を推薦しない
- 見積金額を勝手に記載しない（営業担当に確認を促す）
- 契約条件について法的アドバイスをしない
- 顧客の機密情報（売上、人員数等）を外部に漏らさない

# 注意事項
- 数値データは必ず「要確認」と付記する
- 最終判断はユーザーに委ねる
```

### Bad Example

```markdown
変なことは言わない。
→ 具体性がない
```

---

# 評価例: 出力形式

### Good Example

```markdown
# 出力形式

## 提案書構成
1. タイトル（30文字以内）
2. 課題の整理（3項目、各50文字以内）
3. 提案内容（5項目、各100文字以内）
4. 期待効果（定量2項目、定性2項目）
5. 次のステップ（3項目）

## 文体ルール
- です・ます調
- 専門用語には補足説明を付ける
- 箇条書きを活用
```

# Part 4: よくある問題と対処法

---

# 問題1: 抽象的すぎる

### 症状

```markdown
# 基本動作
ユーザーをサポートします。
```

### 対処法

```markdown
【改善依頼】
基本動作をより具体的にしてください。
具体的には：
- 何を入力として受け取るか
- どんな処理を行うか
- 何を出力するか
を明記してください。
```

---

# 問題2: 構成要素の欠落

### 症状

```markdown
# Instructions
ペルソナと基本動作のみ記載
→ 対話フロー、禁止事項、出力形式が欠落
```

### 対処法

```markdown
【改善依頼】
以下の要素が不足しています。追加してください：
- 対話フロー（Phase 1/Phase 2の構造）
- 禁止事項・注意点
- 出力形式ルール
```

---

# 問題3: 企画書との不整合

### 症状

```markdown
企画書: 「提案書作成」
生成されたInstructions: 「議事録作成」を中心に記載
```

### 対処法

```markdown
【改善依頼】
企画書の内容と整合していません。
特に以下の点を修正してください：
- 目的は「提案書作成」です
- 対象業務は「営業活動」です
企画書を再度読み直し、整合するよう修正してください。
```

---

# 問題4: 禁止事項が不十分

### 症状

```markdown
# 禁止事項
特になし
```

### 対処法

```markdown
【改善依頼】
禁止事項が不足しています。以下の観点を追加してください：
- 競合製品への言及
- 価格・見積の自動記載
- 法的アドバイス
- 機密情報の取り扱い
- 決定権限の範囲
```

---

# 問題5: フォーマットが曖昧

### 症状

```markdown
# 出力形式
きれいに整理して出力します
```

### 対処法

```markdown
【改善依頼】
出力形式を具体的にしてください：
- セクション構成（何をどの順で出力するか）
- 各セクションの文字数目安
- 使用する文体（です・ます調 etc.）
- 箇条書き/表/段落の使い分け
```

# 演習の流れ

### 4つのステップ

```
Step 1: 準備（5分）
├── EX-12の企画書を用意
└── EX-13のメタプロンプトを用意

Step 2: 実行（10分）
├── 変数に企画書を挿入
└── AIに投入して結果を取得

Step 3: 評価（10分）
├── 5構成要素のチェック
└── 評価シートに記入

Step 4: 改善（15分）
├── 不足点を特定
├── 改善依頼を送信（最大3回）
└── 最終版を確定
```

# タイムボックス推奨

### 作業時間の目安

| 作業 | 推奨時間 | 上限 | 備考 |
|---|---|---|---|
| 準備（企画書・メタプロンプト確認） | 5分 | 10分 | 手元にあるか確認 |
| 初回生成 | 5分 | 10分 | AIの応答待ち含む |
