# Part 1: なぜ「対話」が必要なのか

---

# 1回のプロンプトの限界

### よくある状況

```
あなた: 「マーケティング戦略を考えて」

AI: （一般的な戦略を出力）

あなた: 「うーん、なんか違う...」
```

**問題点**:
- AIはあなたの状況を完全には理解していない
- 1回で完璧な回答を得るのは難しい
- 「なんか違う」をうまく伝えられない

---

# 対話で精度を高める

### 改善ループのアプローチ

```
あなた: 「マーケティング戦略を考えて」

AI: 「まずいくつか質問させてください。
      ターゲット顧客は？予算は？期間は？」

あなた: 「BtoB、月50万、3ヶ月」

AI: 「こういう戦略はいかがでしょうか？（提案）
      さらに詳しくしたい点はありますか？」

あなた: 「SNS戦略をもっと具体的に」

AI: （SNS戦略を詳細化）
```

---

# 1回 vs 対話型の比較

| 項目 | 1回完結型 | 対話型（改善ループ） |
|---|---|---|
| プロンプト | 長く詳細 | 短くてOK |
| AIの理解度 | 推測に頼る | 質問で確認 |
| 出力精度 | 運次第 | 回を重ねて向上 |
| 修正コスト | 最初からやり直し | 部分的に修正 |
| ユーザー負担 | 最初に全て指定 | 対話で少しずつ |

---

# 改善ループの本質

```
┌─────────────────────────────────────────┐
│                                         │
│   AIが「理解度を上げる努力」をする      │
│                                         │
│   ユーザーは「フィードバック」をする    │
│                                         │
│   → 両者で答えを作り上げていく         │
│                                         │
└─────────────────────────────────────────┘
```

**重要**: AIに「わからないことは聞いて」と指示する

# Part 2: 改善ループの構造

---

# 改善ループの3ステップ

```
┌────────────────────────────────────────────┐
│                                            │
│   Step 1: まとめ（Summarize）              │
│   「私の理解は〇〇です」                   │
│           ↓                                │
│   Step 2: 提案（Propose）                  │
│   「こういう方向性はいかがでしょうか」     │
│           ↓                                │
│   Step 3: 質問（Ask）                      │
│   「さらに〇〇について教えてください」     │
│           ↓                                │
│   （ユーザーの回答）→ Step 1に戻る        │
│                                            │
└────────────────────────────────────────────┘
```

---

# Step 1: まとめ（Summarize）

### AIが「自分の理解」を示す

```
「ここまでの話をまとめると、以下のようになりますね：
 - ターゲット: BtoB向け中小企業
 - 予算: 月50万円
 - 期間: 3ヶ月間のキャンペーン
 - 目的: 新規リード獲得

 この理解で合っていますか？」
```

**効果**:
- 認識のズレを早期発見
- ユーザーが修正しやすい
- 議論の土台が明確になる

---

# Step 2: 提案（Propose）

### AIが「たたき台」を出す

```
「この条件であれば、以下の施策が考えられます：

1. LinkedIn広告 + ホワイトペーパーDL
2. 業界メディアへの記事広告
3. ウェビナー開催 + フォローメール

特に1は費用対効果が高いと思われます。」
```

**効果**:
- 具体的な議論ができる
- 「この方向性は違う」と言いやすい
- 思考の整理を助ける

---

# Step 3: 質問（Ask）

### AIが「次に必要な情報」を聞く

```
「より良い提案のために、いくつか質問させてください：

1. 過去に効果があった施策はありますか？
2. 競合他社はどんな施策を行っていますか？
3. 社内のコンテンツ制作リソースはありますか？」
```

**効果**:
- ユーザーが見落としていた観点を提示
- 対話を自然に継続
- より深い情報を引き出す

---

# 改善ループの図解

```
      ┌──────────────────────────────────┐
      │                                  │
      │    ┌─────────┐                   │
      │    │ まとめ  │ ← 理解を確認      │
      │    └────┬────┘                   │
      │         ↓                        │
      │    ┌─────────┐                   │
      │    │  提案   │ ← たたき台を出す  │
      │    └────┬────┘                   │
      │         ↓                        │
      │    ┌─────────┐                   │
      │    │  質問   │ ← 次の情報を求める│
      │    └────┬────┘                   │
      │         ↓                        │
      │   [ユーザーの回答]               │
      │         │                        │
      └─────────┴────────────────────────┘
              ↑
           繰り返し
```

# Part 3: 改善ループのプロンプト設計

---

# 改善ループ型プロンプトの基本形

```
あなたは〇〇（ペルソナ）です。

私が〇〇（タスク）を達成できるよう支援してください。

### 動作ルール
以下のサイクルを繰り返してください：

1. **まとめ**: 私が伝えた内容を箇条書きで整理
2. **提案**: 現時点でのアウトプット案を提示
3. **質問**: より良くするために必要な情報を2-3個質問

私が「完了」と言うまで、このサイクルを続けてください。

### 最初のアクション
まず、〇〇について質問から始めてください。
```

---

# 例: 企画書作成の改善ループ

```
あなたは経験豊富な企画コンサルタントです。

私が企画書を作成できるよう支援してください。

### 動作ルール
以下のサイクルを繰り返してください：

1. **まとめ**: 私が伝えた企画の概要を箇条書きで整理
2. **提案**: 企画書のドラフト（または一部）を提示
3. **質問**: 企画を具体化するための質問を2-3個

私が「これでOK」と言うまで、このサイクルを続けてください。

### 最初のアクション
まず、どんな企画を考えているか教えてください。
```

---

# 動作ルールのカスタマイズ

### シーンに応じて調整する

| シーン | まとめ | 提案 | 質問 |
|---|---|---|---|
| 企画立案 | 要点整理 | 企画案 | 差別化ポイント等 |
| 文章作成 | 構成確認 | ドラフト | トーン・読者等 |
| 問題解決 | 状況整理 | 解決策候補 | 制約条件等 |
| 学習支援 | 理解度確認 | 説明・例示 | 疑問点等 |

---

# 終了条件を明確にする

### 良い例
```
私が「完了」と言うまで、このサイクルを続けてください。
```

```
3回のイテレーション後、最終版を出力してください。
```

### 悪い例
```
満足いくまで続けてください。
（→ AIが勝手に判断してしまう）
```

```
良いものができたら終わり。
（→ 曖昧で終わらない）
```

---

# イテレーション数の目安

| 目的 | 推奨イテレーション数 |
|---|---|
| 簡単な確認・修正 | 1-2回 |
| 企画・文章のブラッシュアップ | 3-5回 |
| 複雑な問題解決 | 5-10回 |
| 深い分析・調査 | 必要に応じて |

**ポイント**: 最低3回は回すことで改善効果を実感できる

# Part 4: 改善ループの実践テクニック

---

# テクニック1: 具体的なフィードバック

### 悪い例
```
ユーザー: 「もっと良くして」
AI: （何を改善すべきかわからない）
```

### 良い例
```
ユーザー: 「提案2のSNS戦略について、
          具体的な投稿頻度と内容例を追加して」
AI: （具体的に改善できる）
```

---

# テクニック2: 部分的な承認

### 効率的な進め方

```
ユーザー: 「1と2はOK。3だけ修正して。
          3の〇〇の部分を△△に変えて」
```

**メリット**:
- 全体をやり直さなくて良い
- 何が良くて何が悪いかAIが学習
- 時間の節約

---

# テクニック3: 方向転換の指示

### 途中で方針を変えたくなったら

```
ユーザー: 「やっぱりBtoBではなくBtoCで考えたい。
          最初からやり直して」
```

または

```
ユーザー: 「今の方向性は良いけど、
          もっとコスト重視のプランも見たい。
          別案として出して」
```

---

# テクニック4: 視点を変える質問

### AIに新しい観点を求める

```
ユーザー: 「この企画の弱点は何だと思う？」
```

```
ユーザー: 「競合がこの企画を見たら、
          どう対抗してくると思う？」
```

```
ユーザー: 「この企画を上司に反対されたら、
          どう説得する？」
```

---

# 改善ループの対話例

```
【1回目】
AI: まとめ「BtoB向け新規リード獲得施策ですね」
AI: 提案「LinkedIn広告を中心に...」
AI: 質問「過去の成功施策は？予算配分の優先順位は？」

ユーザー: 「過去はセミナーが効果あった。予算はコンテンツ重視で」

【2回目】
AI: まとめ「セミナー経験あり、コンテンツ重視ですね」
AI: 提案「ウェビナー＋ホワイトペーパーの2本柱で...」
AI: 質問「ウェビナーのテーマは？ホワイトペーパーの既存資産は？」

ユーザー: 「テーマは業界トレンド。既存資料あり」

【3回目】
AI: まとめ「トレンドテーマのウェビナー＋既存資料活用」
AI: 提案「具体的なスケジュールと役割分担を...」
```

---

# プロンプト方式の使い分け

### どの方式を選ぶか？

| 方式 | 適した場面 | 目安時間 |
|---|---|---|
| **1回完結型** | 簡単なタスク、答えが明確 | 〜5分 |
| **改善ループ** | 複雑なタスク、探索的 | 15〜30分 |
| **2段階** | 要件が曖昧、成果物が大きい | 30分〜 |

### タイムボックスの設定

```
改善ループ: 最大5回、または30分で打ち切り
2段階: 要件定義15分、実行15分を目安
```

**→ 無限に続けない。時間を決めて進める**

# 3-1 まとめ

### 改善ループとは
- 「まとめ→提案→質問」を繰り返す対話型プロンプト
- 1回で完璧を目指さず、対話で精度を上げる

### 設計のポイント
- 3ステップを明確に指示
- 終了条件を明示
- 最低3回のイテレーションを推奨

### フィードバックのコツ
- 具体的に伝える
- 部分的に承認する
- 新しい視点を求める

# EX-05: 改善ループ実践【必須】

**課題内容**:
改善ループ型プロンプトを作成し、3回以上のイテレーションを実行する

**成果物**:
1. 改善ループ型プロンプト
2. 3回以上のイテレーション記録（対話ログ）
3. 改善の軌跡分析（何がどう変わったか）

**評価基準**:
- プロンプトに「まとめ→提案→質問」の構造がある
- 3回以上のイテレーションがある
- 各イテレーションで改善が見られる

---

# EX-05 の取り組み方

### ステップ1: テーマを決める
- 企画書、提案書、文章作成など
- 「1回では完成しないもの」を選ぶ

### ステップ2: 改善ループ型プロンプトを作成
- テンプレートを参考に
- 3ステップを含める

### ステップ3: 実行して記録
- 3回以上対話を続ける
- 各イテレーションのAI出力を記録

### ステップ4: 分析
- 何がどう改善されたかをまとめる
