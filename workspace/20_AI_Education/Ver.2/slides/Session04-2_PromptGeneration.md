# Part 1: AIにプロンプトを書いてもらう

---

# なぜAIにプロンプトを書かせるのか？

### 自分で書く場合の課題

| 課題 | 具体例 |
|---|---|
| 構造化が難しい | どこから書き始めればいいか迷う |
| 抜け漏れが起きる | 必要な観点を忘れがち |
| 言い回しに悩む | AIに伝わる表現がわからない |
| 時間がかかる | 試行錯誤に時間を費やす |

---

# AIにプロンプトを書かせるメリット

### メタ的なAI活用

| メリット | 効果 |
|---|---|
| **構造化が自動** | AIが最適な構造を提案 |
| **網羅性が高い** | 必要な観点を漏らさない |
| **表現が最適化** | AIに伝わる言い回しになる |
| **時間短縮** | 数分でプロンプトが完成 |

**→ 「プロンプトを書くプロンプト」= メタプロンプトへの入口**

---

# AIへの依頼の基本形

```
以下の仕様のプロンプトを作成してください。

【目的】
〇〇を作成する

【動作】
1. Phase 1: 要件定義（改善ループで確認）
2. Phase 2: 成果物生成（改善ループで完成まで）

【遷移条件】
- 「OK」でPhase 1 → Phase 2に移行
- 「完了」でセッション終了

【出力形式】
そのままChatGPTにコピペして使えるプロンプト
```

# ステップ1: 仕様を伝える

### 依頼例: 議事録作成プロンプト

```
以下の仕様のプロンプトを作成してください。

【目的】
会議の議事録を作成する

【動作】
1. Phase 1: 会議の概要・参加者・議題を確認（改善ループ）
2. Phase 2: 議事録を作成（改善ループで完成まで）

【遷移条件】
- 「OK」でPhase 1 → Phase 2に移行
- 「完了」でセッション終了

【出力形式】
そのままChatGPTにコピペして使えるプロンプト
```

---

# ステップ2: 生成されたプロンプトを確認

### AIが生成したプロンプト例

```
あなたは経験豊富なビジネスライターです。
会議の議事録を作成します。

【Phase 1: 情報収集】
以下のサイクルを繰り返してください：
1. まとめ: 聞き取った情報を整理
2. 提案: 議事録の構成案を提示
3. 質問: 不足情報を確認

確認事項：会議名、日時、参加者、議題、決定事項、アクション

「OK」と言われたらPhase 2に進んでください。

【Phase 2: 議事録作成】
（続く...）
```

# ステップ4: 必要に応じて修正依頼

### 修正依頼の例

```

2. Phase 2に「機密情報の確認」を追加
3. 出力形式を「表形式」に変更

修正版のプロンプトを出力してください。
```

# ステップ5: 動作確認

### 生成されたプロンプトをテスト

1. 新しいチャットを開く

4. 期待通り動作するか確認

### 確認ポイント
- Phase 1で要件定義が対話的に進むか
- 「OK」でPhase 2に移行するか
- Phase 2で成果物が生成されるか
- 「完了」でセッションが終わるか

# Part 3: よくある失敗と対策

---

# 失敗パターン1: フェーズ遷移しない

### 問題
- 「OK」と言ってもPhase 2に移行しない
- Phase 1のループが続いてしまう

### 対策
```
【明確な遷移指示を追加】
ユーザーが「OK」「進めて」「作成開始」と言ったら、
Phase 1を終了してPhase 2に移行してください。
それ以外の場合はPhase 1を続けてください。
```

---

# 失敗パターン2: 改善ループが回らない

### 問題
- 一発で成果物を出力して終わってしまう
- 修正の提案がない

### 対策
```
【改善ループの明確化】
成果物を出力した後、必ず以下を行ってください：
1. 現在の成果物の概要を説明
2. 改善できそうな点を2-3個提案
3. ユーザーに修正の希望を確認

ユーザーが「完了」と言うまで、このサイクルを続けてください。
```

---

# 失敗パターン3: 品質基準が守られない

### 問題
- Phase 1で確認したDoDが反映されない
- 成果物が要件を満たしていない

### 対策
```
【品質基準の再確認を追加】
Phase 2で成果物を提示する際、以下を確認してください：
- 確定した要件をすべて満たしているか
- 品質基準（DoD）を達成しているか
- 不足がある場合は、その旨を報告
```

# 実践例: 提案書作成プロンプトを生成

### ステップ1: AIへの依頼

```
以下の仕様のプロンプトを作成してください。

【目的】
クライアント向け提案書を作成する

【動作】
1. Phase 1: 提案内容・ターゲット・差別化ポイントを確認
   （改善ループで深掘り）
2. Phase 2: 提案書を作成
   （改善ループで完成まで）

【追加要件】
- 品質基準（DoD）の確認を含める
- 機密情報の取り扱い注意を含める
```

---

# 実践例: 生成されたプロンプト

```
あなたは提案書作成のプロフェッショナルです。

【Phase 1: 要件定義】
以下のサイクルを繰り返してください：
1. まとめ: ヒアリング内容を整理
2. 提案: 提案書の方向性を提示
3. 質問: 詳細を確認

確認事項：
- 提案先（業種・規模・課題）
- 提案内容（商品/サービス）
- 競合との差別化ポイント
- 予算感・決裁者
- 品質基準（何をもって完成とするか）

⚠️ 機密情報は入力しないでください

「OK」と言われたらPhase 2に進んでください。
```

---

# 実践例: 生成されたプロンプト（続き）

```
【Phase 2: 提案書作成】
確定要件に基づいて提案書を作成してください。

作成後、以下のサイクルを繰り返してください：
1. まとめ: 現在の提案書の構成を説明
2. 提案: 改善できる点を提示
3. 質問: 修正の方向性を確認

品質チェック：
- 確定要件をすべて満たしているか確認
- 品質基準（DoD）を達成しているか確認
- 不足があれば報告

「完了」と言われるまで続けてください。
```

# EX-08: AIにプロンプトを書かせる

### 課題内容

AIにプロンプトを生成させ、生成されたプロンプトが
期待通りに動作することを確認してください。

### 提出物
1. AIへの依頼（プロンプト生成の仕様）
2. 生成されたプロンプト
3. 動作確認ログ（生成プロンプトを使った対話）

5. 振り返り（AIにプロンプトを書かせて気づいたこと）

---

# EX-08 取り組みのポイント

### 成功のコツ

1. **仕様を明確に伝える**
   - 目的、動作、遷移条件、出力形式を明記

2. **生成されたプロンプトを必ずテストする**
   - 新しいチャットでコピペして動作確認

3. **問題があれば修正依頼する**
   - 具体的な修正内容を伝える

4. **評価観点を持つ**
   - フェーズ遷移、改善ループ、品質基準
