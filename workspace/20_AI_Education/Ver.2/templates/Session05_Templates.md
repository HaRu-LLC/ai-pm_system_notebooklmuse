# Session 5 テンプレート集

AIポリテラシー育成プログラム Ver.2
Phase 2 / Week 5

---

## 使い方

1. 受講アプリからこのテンプレートを**ダウンロード**してください
2. 必要なテンプレートを選んで、〇〇の部分を自分の内容に置き換えてください
3. 完成したらChatGPTに入力して実行してください
4. 課題を受講アプリで提出してください

## テンプレート1: 汎用プロンプトテンプレート（EX-09用）

### 基本構造（変数化されたテンプレート）

```
あなたは{役割}です。

【Phase 1: 要件定義】
以下のサイクルを繰り返してください：
1. まとめ: 私が伝えた内容を箇条書きで整理
2. 提案: {成果物}の構成案を提示
3. 質問: 以下の観点について確認
   - {確認観点}
   - 品質基準（完成とみなす条件）

私が「OK」「進めて」と言ったら、Phase 2に進んでください。
それ以外の場合はPhase 1を続けてください。

【Phase 2: {目的}】
確定した要件に基づいて、{成果物}を作成してください。
{制約}を守ってください。

作成後、以下のサイクルを繰り返してください：
1. まとめ: 現在の成果物の概要を説明
2. 提案: 改善できる点を2-3個提示
3. 質問: 修正の方向性を確認

品質チェック：
- 確定要件をすべて満たしているか
- 品質基準（DoD）を達成しているか

私が「完了」と言うまで、このサイクルを続けてください。

⚠️ 注意事項
- 機密情報は入力しないでください
- 数値・事実は必ず確認してください
```

---

### 変数定義シート

| 変数名 | 説明 | 入力ガイド |
|---|---|---|
| **{役割}** | AIが演じるペルソナ | 「〇〇のプロフェッショナル」「〇〇のエキスパート」など |
| **{目的}** | このセッションで達成すること | 「〇〇を作成」「〇〇を分析」など動詞で終わる |
| **{成果物}** | 最終的なアウトプット | 具体的な形式（A4 3ページ、10項目のリスト等） |
| **{制約}** | 守るべきルール | 文字数、フォーマット、禁止事項など |
| **{確認観点}** | Phase 1で確認する項目 | 5-7個程度、目的に応じて設定 |

> ⚠️ **注意**: DoD（品質基準）とガードレールは**固定項目**です。変数化せず、テンプレートに固定で含めてください。

---

### 共通DoD（品質基準）プリセット

どの用途でも使える最低限の完成基準です。用途固有の基準があれば追加してください。

```
【共通DoD（品質基準）】
1. 目的達成度: 依頼した目的を満たしているか
2. 完成度: 必要な要素がすべて揃っているか
3. 実用性: そのまま業務で使える状態か
4. 正確性: 事実・数値に誤りがないか
```

## テンプレート2: メタプロンプト（EX-10用）

### 基本構造

```
あなたはプロンプト設計のエキスパートです。
ユーザーが達成したい目的を聞き、最適なプロンプトを生成してください。

【対話フロー】
1. ユーザーの目的を確認
2. 以下の変数を推論し、提案
   - {役割}: 目的達成に最適な専門家
   - {目的}: 達成したいこと
   - {成果物}: アウトプットの形式
   - {制約}: 守るべきルール
   - {確認観点}: 要件定義で確認すべき項目（5-7個）
3. ユーザーの承認を得たらプロンプトを生成

【変数設定ルール】
- 推論に自信がない項目はユーザーに確認
- 業界慣行や一般的なベストプラクティスを考慮
- 過度に複雑にせず、実用的な設定を優先

【生成するプロンプトの構造】
- ペルソナ: {役割}
- Phase 1: 要件定義（まとめ→提案→質問のループ）
  - 遷移条件: 「OK」「進めて」でPhase 2へ
  - 確認観点: {確認観点} + 品質基準（DoD）
- Phase 2: 成果物生成（まとめ→提案→質問のループ）
  - 終了条件: 「完了」でセッション終了
  - 制約: {制約}

【ガードレール】
生成するプロンプトには、以下の注意事項を含めてください：
⚠️ 機密情報は入力しないでください
⚠️ AIの出力は事実確認が必要です
```

## 変数化のコツ

### 1. 変数は5-7個程度に抑える

```
✅ 良い例
{役割}, {目的}, {成果物}, {制約}, {確認観点}
→ 埋めやすく、覚えやすい

❌ 悪い例
{役割}, {目的}, {成果物}, {制約}, {確認観点},
{トーン}, {形式}, {長さ}, {対象者}, {除外事項}...
→ 埋めるのが大変、ミスが増える
```

### 2. 変数名は直感的に

```
✅ 良い例
{役割} → 何を入れるか明確
{成果物} → アウトプットだとわかる

❌ 悪い例
{var1}, {param_a} → 何を入れるか不明
```

### 3. デフォルト値を用意する

```
【変数定義】
{役割} = ビジネスコンサルタント（デフォルト）
{確認観点} = 目的、読者、必須要素、制約、優先度（デフォルト）

→ 埋め忘れても動作する
→ 必要に応じてカスタマイズ
```

## 業務移管時のチェックリスト

### テンプレート使用前に確認

- [ ] **機密区分**: 社外秘情報をAIに入力していないか
- [ ] **事実確認**: AIが出力した数値・事実を元データで確認したか
- [ ] **責任者**: 最終的な承認・責任者は明確か
- [ ] **再利用可否**: この成果物を他者が使ってよいか確認したか
- [ ] **品質基準**: 定義したDoDを満たしているか

### テンプレート固有のチェック

- [ ] **変数設定**: すべての変数が適切に埋められているか
- [ ] **用途適合**: 選んだ変数設定がこの用途に適しているか
- [ ] **テスト済み**: 生成されたプロンプトが正しく動作するか

---

## 困ったときは

- **変数の設定がわからない** → サンプル変数設定を参考にする
- **テンプレートが動作しない** → ライト版テンプレートで試す
- **メタプロンプトの推論が不正確** → 自分で変数を指定する
- **チームで統一できない** → プリセットを定義して共有する
