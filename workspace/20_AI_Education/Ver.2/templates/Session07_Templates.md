## EX-13: Instructions生成メタプロンプト作成

### 課題の目的

Session 6で作成したGPTs企画書を入力として、Instructions を生成するメタプロンプトを作成する。

### 提出物

1. Instructions生成メタプロンプト（変数化されたもの）
2. 変数定義シート（各変数の説明）
3. 変換マッピング表（企画書項目→Instructions要素）
4. 振り返り（なぜ手書きではなくメタプロンプトを使うのか）

### タイムボックス推奨

| 作業 | 推奨時間 | 上限 | 備考 |
|---|---|---|---|
| メタプロンプト設計 | 20分 | 30分 | テンプレートをベースに |
| 変数定義シート作成 | 10分 | 15分 | 3-5変数に絞る |
| 変換マッピング作成 | 10分 | 15分 | 6項目の対応 |
| 動作確認（簡易テスト） | 10分 | 15分 | 企画書を入力して確認 |
| 振り返り記入 | 10分 | 15分 | |
| **合計** | **60分** | **90分** | |

> **ポイント**: 完璧を目指さず、まず動くものを作る。改善はEX-14で行う

### テンプレート3: 変換マッピング表

企画書の各項目がInstructionsのどの要素に対応するかを整理します。

| # | 企画書の項目 | Instructions要素 | 変換のポイント |

## EX-14: GPTs Instructions v1（AI生成）

### 課題の目的

EX-13で作成したメタプロンプトを実行し、実際にInstructionsを生成する体験をする。

### 提出物

1. 使用したメタプロンプト（EX-13のもの）
2. 入力した企画書（EX-12のもの）
3. AIとの対話ログ（Phase 1 + Phase 2）
4. 生成されたGPTs Instructions v1
5. 自己評価シート（5観点×20点）
6. 振り返り（改善ループで気づいたこと）

### タイムボックス推奨

| 作業 | 推奨時間 | 上限 | 備考 |
|---|---|---|---|
| 準備（企画書・メタプロンプト確認） | 5分 | 10分 | 手元にあるか確認 |
| 初回生成 | 5分 | 10分 | AIの応答待ち含む |
| 評価・チェック | 10分 | 15分 | 5構成要素を確認 |
| 改善ループ | 15分 | 20分 | 最大3回のイテレーション |
| 振り返り記入 | 10分 | 15分 | |
| **合計** | **45分** | **70分** | |

> **ポイント**: 改善ループが終わらない場合は3回で完了し、課題を振り返りに記録

```
## 2. 5観点評価

| 観点 | 配点 | 自己評価 | 根拠・コメント |
|---|---|---|---|

| 実用性 | 20点 | ____点 | |
| 安全性 | 20点 | ____点 | |
| **合計** | **100点** | ____点 | |

### 評価基準（参考）

#### 完全性（7構成要素がすべて含まれているか）
- 20点: 7要素すべて詳細に記載
- 15点: 7要素すべてあるが一部簡略
- 10点: 5-6要素のみ
- 5点: 4要素以下

#### 具体性（抽象的でなく具体的な記述か）
- 20点: 動詞で始まる具体的な動作、数値目安あり
- 15点: 具体的だが一部曖昧
- 10点: 半分程度が抽象的
- 5点: ほとんど抽象的

#### 一貫性（企画書の内容と整合しているか）
- 20点: 企画書の内容が正確に反映
- 15点: 概ね反映されているが一部ずれ
- 10点: 主要部分にずれあり
- 5点: 企画書と大きく乖離

#### 実用性（そのままGPTsに設定できる形式か）
- 20点: コピペで設定可能
- 15点: 軽微な修正で設定可能
- 10点: かなりの修正が必要
- 5点: 書き直しが必要

#### 安全性（優先順位・例外処理・幻覚対策が適切か）
- 20点: 優先順位明記＋例外処理5観点＋幻覚対策あり
- 15点: 優先順位明記＋例外処理4観点
- 10点: 優先順位または例外処理のどちらかのみ
- 5点: 両方とも不十分

## 3. 7構成要素チェック

| # | 要素 | 有無 | 詳細度 | コメント |
|---|---|---|---|---|
| 1 | ペルソナ定義 | □あり □なし | 高/中/低 | |
| 2 | 基本動作指示 | □あり □なし | 高/中/低 | |
| 3 | 対話フロー | □あり □なし | 高/中/低 | |
| 4 | **優先順位** | □あり □なし | 高/中/低 | 安全性>正確性>有用性>効率性 |
| 5 | 禁止事項・注意点 | □あり □なし | 高/中/低 | |
| 6 | **例外処理** | □あり □なし | 高/中/低 | 幻覚対策含む |
```

## 補足資料

### Instructions設計でよくある失敗チェックリスト

Instructionsを最終確認する際に使用してください。

```
【Instructionsチェックリスト】

□ ペルソナが具体的に設定されているか
  NG: 「AIアシスタント」 → OK: 「10年経験の営業マネージャー」

□ 基本動作が動詞で始まっているか
  NG: 「サポートを行います」 → OK: 「ヒアリングする」「生成する」

□ 対話フローがPhase構造になっているか
  NG: 「質問してから作成」 → OK: 「Phase 1→Phase 2」

□ 禁止事項が5観点をカバーしているか
  必須: 競合/価格/法務/機密/権限

□ 出力形式に文字数・構成が明記されているか
  NG: 「きれいに」 → OK: 「1文60文字以内、5セクション構成」

□ 例外処理が記載されているか
  NG: なし → OK: 「範囲外の質問には〜と回答」
```

### よくある改善依頼パターン集

改善ループで使える依頼文のテンプレートです。

```
【完全性の改善】
「禁止事項・注意点のセクションが不足しています。
以下の5観点を追加してください：
- 競合製品への言及
- 価格・見積の取り扱い
- 法的アドバイスの範囲
- 機密情報の保護
- 決定権限の範囲」

【具体性の改善】
「基本動作指示が抽象的です。
各動作を以下の形式で具体化してください：
- 動詞で始める
- 何を入力として受け取るか明記
- 何を出力するか明記」

【一貫性の改善】
「企画書の内容と整合していません。
特に以下の点を企画書に合わせて修正してください：
- 目的は「〇〇」です
- 対象業務は「△△」です」

【実用性の改善】
「そのままGPTsに設定できる形式にしてください。
- Markdown形式で整理
- セクション見出しを明確に
- コピペ可能な状態に」

【安全性の改善】
「例外処理が不足しています。
以下のシナリオへの対応を追加してください：
- 範囲外の質問への対応
- 機密情報入力時の対応
- 不明確な指示への対応」
```

```
         ↓
         ↓ Session 8の参照
         ↓
Session 8の成果物
├── EX-15: 知識ベース設計メタプロンプト
└── EX-16: 知識ベース v1
         ↓
         ↓ Session 9で統合
         ↓
Session 9の成果物
└── EX-17-18: GPTs統合テスト
         ↓
         ↓ フィードバック反映
         ↓
Session 10の成果物
├── EX-19: Instructions v2（改善版）
└── EX-20: GPTs完成版
```

Instructions v1は大切に保管し、次回以降のセッションで活用してください。
