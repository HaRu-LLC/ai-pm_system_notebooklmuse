## EX-11: GPTs企画メタプロンプト作成

### 課題の目的

GPTs企画書を生成するためのメタプロンプトを作成し、メタプロンプトの活用方法を習得する。

### 提出物

1. GPTs企画メタプロンプト（変数化されたもの）
2. 変数定義シート（各変数の説明）
3. 動作確認ログ（メタプロンプトを使った対話）
4. 振り返り（メタプロンプトでGPTs企画を生成するメリット）

### タイムボックス推奨

| 作業 | 推奨時間 | 上限 | 備考 |
|---|---|---|---|
| メタプロンプト設計 | 20分 | 30分 | テンプレートをベースに |
| 変数定義シート作成 | 10分 | 15分 | 5-7変数に絞る |
| 動作確認（Phase 1） | 15分 | 20分 | 最大3回のイテレーション |
| 動作確認（Phase 2） | 15分 | 20分 | 最大3回のイテレーション |
| 振り返り記入 | 10分 | 15分 | |
| **合計** | **60分** | **90分** | |

> **ポイント**: 改善ループが終わらない場合は、3回を上限として「完了」し、振り返りに課題を記録する

---

### テンプレート2: 変数定義シート

| 変数名 | 説明 | デフォルト値 |
|---|---|---|
| {役割} | AIが演じるペルソナ | GPTs設計コンサルタント |
| {確認観点} | Phase 1で確認する項目リスト | 対象業務・課題・ユーザー・リソース・成功基準 |
| {出力形式} | 企画書の構成項目 | 6項目構成（下記参照） |

#### {確認観点} のデフォルト値

```
- 対象業務の詳細
- 現状の課題・ボトルネック
- 想定ユーザーとその人数
- 活用可能な既存リソース
- 達成したい成功基準
```

#### {出力形式} のデフォルト値

```
1. GPTs名
2. 目的・解決する課題
3. 機能概要（3-5項目）
4. 想定される対話フロー
5. 必要な知識ベース
6. 成功指標
```

> ⚠️ **注意**: DoD（品質基準）とガードレールは**固定項目**です。変数化せず、テンプレートに固定で含めてください。

## EX-12: GPTs企画書（AI生成）

### 課題の目的

EX-11で作成したメタプロンプトを実際に使用し、GPTs企画書をAIに生成させる体験をする。

### 提出物

1. 使用したメタプロンプト（EX-11のもの）
2. 入力した業務情報
3. AIとの対話ログ（Phase 1 + Phase 2）
4. 生成されたGPTs企画書
5. 振り返り（企画書の評価・改善点）

## 補足資料

### GPTs企画でよくある失敗チェックリスト

企画書を最終確認する際に使用してください。

```
【企画書チェックリスト】

□ GPTs名が具体的でわかりやすいか
  NG: 「営業支援」 → OK: 「提案書ドラフター Pro」

□ 目的が数値化されているか
  NG: 「効率化する」 → OK: 「作成時間を50%削減する」

□ 機能が3-5個に絞られているか
  NG: 10個の機能リスト → OK: 主要機能5個にフォーカス

□ 対話フローが自然か
  NG: 複雑な条件分岐 → OK: 直線的な3-5ステップ

□ 知識ベースが現実的か
  NG: 「すべての過去資料」 → OK: 「過去1年の成功事例10件」

□ 成功指標が測定可能か
  NG: 「満足度向上」 → OK: 「満足度スコア3.5→4.0」
```
