## 1. 制作概要

### 1.1 制作規模

| 項目 | 数量 | 備考 |
|---|:---:|---|
| 総セッション数 | 14 | Session 0（オリエンテーション）+ Session 1-13 |
| 総動画本数 | 約30本 | Session 0: 3本、Session 1: 2本、Session 2-13: 各約2本 |
| 総収録時間 | 約15時間 | パート単位での短縮版 |
| 動画1本あたり | 15-25分 | パートにより異なる |

> **構成変更**: 従来の「導入/本編①/本編②/まとめ」4パート構成から、各Sessionで2-3パート構成に変更。実習課題は各パートの最後に配置。

### 1.2 制作体制

| 役割 | 担当 | 責務 |
|---|---|---|
| 講師・収録 | 井戸 | 講義・デモ収録 |
| 企画・構成 | AI-PgM | カリキュラム設計・台本レビュー |
| 編集 | 外注/自動化 | カット編集・テロップ・エフェクト |
| 品質チェック | AI-PgM | 最終確認・修正指示 |

### 1.3 制作フロー（1セッションあたり）

```
企画・構成（2日）→ 台本作成（2日）→ 収録（1日）→ 編集（3日）→ レビュー（1日）→ 公開準備（1日）
```

**1セッション制作期間**: 約10営業日（2週間）

## 4. 動画仕様

### 4.1 技術仕様

| 項目 | 仕様 |
|---|---|
| 解像度 | 1920×1080 (Full HD) |
| フレームレート | 30fps |
| コーデック | H.264 |
| 音声 | AAC, 48kHz, ステレオ |
| 形式 | MP4 |
| ファイルサイズ目安 | 500MB〜1GB/本 |

### 4.2 画面構成

| 要素 | 仕様 |
|---|---|
| メイン画面 | スライド（16:9） |
| 講師映像 | 右下または左下ワイプ（任意） |
| テロップ | 下部1/5、重要ポイント強調 |
| ロゴ | 右上固定（HaRuロゴ） |
| 進捗バー | 下部、セッション内進捗表示 |

### 4.3 ファイル命名規則

```
AIPoliteracy_S{セッション番号}-{パート番号}_{パート名}.mp4
```

**例**:
- `AIPoliteracy_S00-1_ProgramOverview.mp4`（Session 0-1: プログラム概要）
- `AIPoliteracy_S00-2_LearningMethod.mp4`（Session 0-2: 学習方法）
- `AIPoliteracy_S00-3_Orientation.mp4`（Session 0-3: ツール準備）
- `AIPoliteracy_S01-1_PromptBasics_Theory.mp4`（Session 1-1: 理論編）
- `AIPoliteracy_S01-2_PromptBasics_Practice.mp4`（Session 1-2: 実践編）

> **構成方針**: 各パートで「理論→実習案内」または「実践→実習案内」のセットで完結。デモ動画は廃止し、Before/After例やパターン紹介をスライド内で提示。

```markdown

## メタ情報
- 収録予定日: YYYY-MM-DD
- パート数: 2
- 講師: 井戸

---

## Part {番号}-1: {パート1タイトル}（約20分）

### スライド1: タイトル
**ナレーション**:
「皆さん、こんにちは。Session{番号}、Part 1へようこそ。」

**画面**: タイトルスライド

### スライド2: 本パートの学習目標
```

## 6. 品質チェックリスト

### 6.1 収録前チェック

- [ ] 台本レビュー完了
- [ ] スライド作成完了
- [ ] デモ環境準備完了
- [ ] 機材テスト完了（マイク、カメラ、画面録画）
- [ ] 収録環境整備（照明、背景、騒音対策）

### 6.2 収録後チェック

- [ ] 音声品質確認（ノイズ、音量レベル）
- [ ] 映像品質確認（ピント、明るさ）
- [ ] 内容確認（言い間違い、情報の正確性）
- [ ] 時間確認（各パートの尺）

### 6.3 編集後チェック

- [ ] カット編集確認（不要部分削除）
- [ ] テロップ確認（誤字脱字、タイミング）
- [ ] BGM/SE確認（音量バランス）
- [ ] ロゴ・ウォーターマーク確認
- [ ] エンコード品質確認

### 6.4 公開前チェック

- [ ] 全パート通し視聴
- [ ] リンク・URL確認
- [ ] サムネイル作成
- [ ] 概要欄・説明文作成
- [ ] アクセス権限設定（限定公開）
- [ ] **Webアプリへの動画URL登録**
- [ ] Webアプリ上での動画再生確認

### 6.5 視聴チェックイン連携

> [!IMPORTANT]
> 助成金申請には視聴証跡が必要。視聴チェックインはWebアプリで統合管理。

**対応方法**:
- 各動画の概要欄/終了画面に「受講アプリでチェックインしてください」と案内
- **Webアプリのセッション詳細ページ**に動画を埋め込み、下部に「視聴完了」ボタンを設置
- 「動画視聴後、下の『視聴完了』ボタンを押してください」と案内
- チェックイン記録はDB（viewing_logsテーブル）に自動保存

### 7.3 配信プラットフォーム

| プラットフォーム | 用途 | 備考 |

> **注意**: Notion/Google Sitesは使用しない。専用WebアプリでコンテンツとUIを一元管理する。

## 8. リスクと対策

| リスク | 影響度 | 対策 |
|---|:---:|---|
| 収録遅延 | 高 | バッファ期間確保、並行作業可能な体制 |
| 編集工数増大 | 中 | AI編集ツール活用、テンプレート化 |
| 品質問題による再収録 | 高 | 事前チェックリスト徹底、テスト収録 |
| 機材トラブル | 中 | 予備機材準備、クラウドバックアップ |
| 講師体調不良 | 高 | スケジュールバッファ、優先度に基づく調整 |
