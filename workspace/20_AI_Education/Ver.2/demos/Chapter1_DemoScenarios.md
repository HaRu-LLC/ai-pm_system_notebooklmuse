## デモ実施ガイドライン（参考）

### 基本方針

1. **再現性**: 受講者が同じ操作で同様の結果を得られること
2. **段階的改善**: Before→Afterの変化を明確に見せる
3. **失敗も見せる**: 悪い例を先に見せて問題を実感させる
4. **リアルタイム**: 事前録画ではなくライブ感を大切に
5. **解説付き**: 操作しながら「なぜそうするか」を説明

### 使用ツール

| ツール | 用途 | 備考 |
|---|---|---|
| ChatGPT (GPT-4) | メインデモ | Plus/Team契約必須 |
| Claude | 比較デモ（任意） | 応答の違いを見せる |
| 画面録画 | バックアップ | 万が一の障害時用 |

### デモ前チェックリスト

- [ ] ChatGPTにログイン済み
- [ ] 新規チャットを開始（履歴の影響を排除）
- [ ] 画面共有の準備完了
- [ ] プロンプトをクリップボードにコピー済み
- [ ] バックアップ録画を再生可能な状態に

## Session 1: プロンプトの基本構造

### Demo 1-1: 悪いプロンプト vs 良いプロンプト（導入）

**目的**: プロンプトの質が応答に影響することを体感させる

#### Step 1: 悪いプロンプトを実行

**操作手順**:
1. ChatGPTで新規チャットを開始
2. 以下のプロンプトを入力

```
いい文章書いて
```

**予想される応答**:
- 「どのような文章をお望みですか？」と質問される
- または曖昧で使えない文章が生成される

**解説ポイント**:
- 「何が書いてほしいか分からない」
- 「AIは心を読めない」
- 「人間に頼むときと同じ - 具体的に伝える必要がある」

#### Step 2: 良いプロンプトを実行

**操作手順**:
1. 同じチャットで以下を入力

```
あなたはビジネスメールの専門家です。

以下の条件でお礼メールを作成してください：

【状況】
- 取引先との初回商談が成功した
- 次回のミーティング日程を調整したい

【条件】
- 200文字程度
- 丁寧な敬語
- 前向きな印象を与える

【出力形式】
件名と本文を分けて出力
```

**予想される応答**:
- 適切な件名と本文が生成される
- 指定した条件を満たしている

**解説ポイント**:
- 「4つの要素が入っている」
- 「指示が具体的だから具体的な答えが返る」
- 「これがプロンプトエンジニアリング」

### Demo 1-2: 4要素の段階的追加（本編②）

**目的**: プロンプトを段階的に改善する過程を見せる

#### Step 1: 指示だけ

```
メールを書いて
```

**解説**: 「何のメールか分からない」

#### Step 2: 指示 + 文脈

```
取引先への商談お礼メールを書いて
```

**解説**: 「目的は分かるが、どんなトーンで書くか不明」

#### Step 3: 指示 + 文脈 + 制約

```
取引先への商談お礼メールを書いて

条件：
- 200文字程度
- 丁寧な敬語で
- 次回のミーティング希望も伝える
```

**解説**: 「だいぶ良くなったが、形式が指定されていない」

#### Step 4: 完成形（4要素すべて）

```
あなたはビジネスメールの専門家です。

取引先への商談お礼メールを作成してください。

【状況】
先週の商談で新規取引の方向性が固まった。
次回は具体的な契約条件を詰めたい。

【条件】
- 200文字程度
- 丁寧な敬語
- 前向きな印象
- 次回ミーティングの希望日程（来週火曜か水曜）を提示

【出力形式】
- 件名
- 本文
- 署名（〇〇株式会社 営業部 山田太郎）
```

**解説**: 「4要素が揃うと、意図通りの出力が得られる」

#### Before/After比較（画面分割で見せる）

| 項目 | Before | After |
|---|---|---|
| プロンプト | 1行 | 15行 |
| 応答品質 | 曖昧 | 具体的・実用的 |
| 修正回数 | 何度も必要 | 1回でほぼ完成 |

### Demo 1-3: よくある失敗パターン（本編②）

**目的**: 典型的な失敗を見せて回避策を学ぶ

#### 失敗パターン①: 曖昧な指示

```
なんかいい感じの企画書作って
```

**問題**: 「いい感じ」の定義がない
**改善**: 具体的な評価基準を示す

#### 失敗パターン②: 情報過多

```
私は〇〇会社の営業部で働いていて、入社5年目で、趣味は釣りで、最近は忙しくて釣りに行けていなくて、でも来月は休みが取れそうで、それはさておき、取引先の田中さんにメールを送りたいんですが、田中さんは50代の男性で、ゴルフが好きで、先日のコンペでは80台で回ったらしく、それで商談の件なんですが...（続く）
```

**問題**: 関係ない情報が多すぎて本題が埋もれる
**改善**: 目的に関連する情報だけに絞る

#### 失敗パターン③: 矛盾する制約

```
500文字以内で、詳細に、全ての観点から網羅的に説明して
```

**問題**: 「短く」と「詳細に」が矛盾
**改善**: 優先順位を明確にするか、制約を調整

#### 失敗パターン④: 出力形式の曖昧さ

```
分かりやすくまとめて
```

**問題**: 「分かりやすい」の基準が人それぞれ
**改善**: 具体的な形式（箇条書き、表など）を指定

#### 失敗パターン⑤: ペルソナなし

```
この文章を添削して
```

**問題**: どの立場で添削するか不明（先生？編集者？友人？）
**改善**: 「あなたはプロの編集者です」などペルソナを設定

## Session 2: 文脈と制約の設計

### Demo 2-1: ペルソナ比較（本編①）

**目的**: ペルソナ設定で応答が大きく変わることを見せる

#### 共通の質問

```
「生成AIを業務で使うべきか？」という質問に答えてください。
```

#### ペルソナA: 楽観的なテック推進派

```
あなたはAI導入を積極的に推進するITコンサルタントです。
新技術に前向きで、効率化のメリットを重視します。

「生成AIを業務で使うべきか？」という質問に答えてください。
300文字程度で回答してください。
```

#### ペルソナB: 慎重なリスク管理者

```
あなたは企業のリスク管理責任者です。
セキュリティとコンプライアンスを最優先に考えます。

「生成AIを業務で使うべきか？」という質問に答えてください。
300文字程度で回答してください。
```

#### ペルソナC: バランス型の経営者

```
あなたは中小企業の経営者です。
コストと効果のバランス、従業員への影響を考慮します。

「生成AIを業務で使うべきか？」という質問に答えてください。
300文字程度で回答してください。
```

**解説ポイント**:
- 3つの応答を並べて比較
- 「同じ質問でもペルソナで答えが変わる」
- 「目的に合ったペルソナを選ぶことが重要」

### Demo 2-2: 制約の有無比較（本編②）

**目的**: 制約の効果を実感させる

#### 制約なしバージョン

```
生成AIの活用事例を教えて
```

**予想される応答**: 長文で網羅的、読みにくい

#### 制約ありバージョン

```
生成AIの業務活用事例を教えてください。

【制約】
- 日本企業の事例に限定
- 3事例まで
- 各事例は100文字以内
- 業種・活用内容・効果を含める
- 表形式で出力
```

**予想される応答**: コンパクトで読みやすい表

**解説ポイント**:
- 「制約がないと発散する」
- 「制約で焦点を絞れる」
- 「形式制約で加工しやすくなる」

### Demo 2-3: 制約の組み合わせ（本編②）

**目的**: 複数の制約を効果的に使う方法を見せる

```
新入社員向けの生成AI研修企画書を作成してください。

【ペルソナ】
あなたは企業研修の専門家です。

【長さの制約】
- 全体で800文字程度

【形式の制約】
- 見出しを使って構造化
- 箇条書きを活用

【トーンの制約】
- 前向きで親しみやすい
- 専門用語は避ける

【内容の制約】
- 研修目的、対象者、カリキュラム概要、期待効果を含める

【禁止事項】
- 技術的な詳細には踏み込まない
- コスト面には言及しない
```

**解説ポイント**:
- 「5種類の制約を組み合わせている」
- 「制約同士が矛盾しないように注意」
- 「禁止事項で余計な情報を防ぐ」

## Session 3: 出力形式の最適化

### Demo 3-1: 同じ内容を5形式で出力（本編①）

**目的**: 出力形式の違いを視覚的に比較

#### 共通のお題

「生成AIのメリット5つ」を以下の形式で出力

#### 形式1: プレーンテキスト

```
生成AIのメリットを5つ、通常の文章形式で説明してください。
```

#### 形式2: 番号付きリスト

```
生成AIのメリットを5つ、番号付きリストで説明してください。
各項目は1文で簡潔に。
```

#### 形式3: 表形式

```
生成AIのメリットを5つ、以下の表形式で出力してください。

| No | メリット | 説明（30文字以内） | 具体例 |
```

#### 形式4: Markdown形式

```
生成AIのメリットを5つ、Markdown形式で出力してください。
- H2見出しで各メリット
- 本文で説明
- 重要キーワードは**太字**
```

#### 形式5: JSON形式

```
生成AIのメリットを5つ、JSON形式で出力してください。

{
  "benefits": [
    {
      "id": 1,
      "title": "メリット名",
      "description": "説明",
      "example": "具体例"
    }
  ]
}
```

**解説ポイント**:
- 5つの出力を並べて表示
- 「同じ内容でも形式で印象が変わる」
- 「用途に応じて選ぶ」
  - プレーン: 読み物として
  - リスト: 箇条書きで整理
  - 表: 比較・一覧表示
  - Markdown: ドキュメント作成
  - JSON: プログラム連携

**目的**: 実務で即使えるメール作成デモ

#### お礼メール

【状況】

**ライブでカスタマイズ**:
- 受講者から状況を聞いてその場でプロンプトを調整

### Demo 3-3: 議事録作成（本編②）

**目的**: メモから議事録を生成するデモ

#### 入力：会議メモ（箇条書き）

```
あなたは議事録作成の専門家です。

以下の会議メモから正式な議事録を作成してください。

【会議メモ】
- 日時: 12/20 14:00-15:00
- 参加: 山田、鈴木、田中、佐藤
- 場所: 会議室A
- 議題: 来期の予算について
- 山田: 前年比10%増を希望
- 鈴木: 厳しいのでは？5%が現実的
- 田中: 新規事業分を別枠で確保したい
- 佐藤: マーケティング費用は削減余地あり
- 結論: 7%増で調整、新規事業は別途検討
- 次回: 1/10に詳細検討

【出力形式】
# 会議議事録

## 基本情報
（表形式）

## 議事内容
（発言者ごとに整理）

## 決定事項
（箇条書き）

## 次回アクション
（担当者と期限を明記）
```

**解説ポイント**:
- 「メモを構造化された議事録に変換」
- 「出力形式を詳細に指定するのがコツ」
- 「自分の業務に合わせてテンプレート化できる」

### Demo 3-4: レポート作成（本編②）

**目的**: 企画書・報告書の骨子作成デモ

```
あなたは企画書作成の専門家です。

以下のテーマで企画書の骨子を作成してください。

【テーマ】
社内の生成AI活用推進プロジェクト

【背景】
- 競合他社がAI活用で成果を出している
- 社内からも「AIを使いたい」という声が増加
- 経営層も関心を持っている

【目的】
社内でのAI活用を段階的に推進し、業務効率化を実現する

【条件】
- A4で2ページ程度の分量感
- 経営層向けの説得力ある構成
- 具体的な数字は仮置きでOK

【出力形式】
1. タイトル
2. エグゼクティブサマリー（3行）
3. 背景・課題
4. 目的・ゴール
5. 実施内容（フェーズ分け）
6. 期待効果（定量・定性）
7. スケジュール
8. 必要リソース
9. リスクと対策
10. 次のステップ
```

**解説ポイント**:
- 「企画書の構成をAIに任せる」
- 「骨子を作ってから肉付けする使い方」
- 「形式を詳細に指定すると漏れがない」
