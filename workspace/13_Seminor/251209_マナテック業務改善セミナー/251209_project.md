## 1. プロジェクト概要

### 1.1 背景と目的

**背景:**
マナテック株式会社の管理本部（経理、総務、人事などバックオフィス部門）において、業務効率化とDX推進が求められている。現状の業務プロセスを可視化し、ボトルネックを特定して改善策を立案・実行する組織風土を醸成する必要がある。

**目的:**
- 管理本部全メンバー（30名程度）が業務プロセスの可視化手法を習得する
- 各チームが自チームの業務フローを見える化し、ボトルネックを特定する
- 実践的なワークショップを通じて、改善アイデアを創出し行動変容を促す
- 「この業務を改善してみよう」という内発的動機を全員に芽生えさせる

### 1.2 プロジェクトゴール（KGI）

| 指標 | 目標値 | 測定方法 |
|---|---|---|
| **ワークショップ完遂率** | 100%（全5チーム） | 各チームが業務フロー見える化シート完成 |
| **改善計画策定率** | 100%（全5チーム） | 各チームが改善計画を発表 |
| **行動変容意欲** | 80%以上 | 事後アンケート「この業務を改善してみようと思う」 |
| **参加者満足度** | 4.0/5.0以上 | 事後アンケート5段階評価 |
| **知識定着度** | 70%以上 | 事後テスト or 振り返りシート |

### 2.3 ワークショップ①詳細設計（120分）

#### フェーズ1: 導入・手法説明（10分）
- 業務プロセス可視化の目的と効果
- 見える化フレームワークの説明
- 模造紙＋付箋の使い方デモンストレーション

#### フェーズ2: 業務フロー見える化ワーク（20分）
**使用ツール:**
- 模造紙（各グループ1枚）
- 付箋（複数色）
- ペン、マーカー

**見える化マトリクス項目:**
| 項目 | 記載内容 |
|---|---|
| 業務名 | タスクの名称 |
| 内容 | 具体的な作業内容 |
| 担当者 | 実施者（役割） |
| 入力（インプット） | 必要な情報・資料 |
| 出力（アウトプット） | 成果物 |
| 判断ポイント | 意思決定が必要な箇所 |
| 苦労している点 | 課題・困りごと |
| 工夫している点 | ノウハウ・ベストプラクティス |

**進行方法:**
1. 各メンバーが自分の担当業務を付箋に記入
2. グループ内で業務フローの順序を整理
3. マトリクスに沿って主要な項目を記入（詳細化は後ほど）

#### フェーズ3: ワールドカフェ実施（30分）
フェーズ2で作成した業務フローをもとに、知識の相互学習を促進する**ワールドカフェ**を実施します。

**ワールドカフェ構成:**
- **ラウンド1（15分）**: 各グループから1名が「ホスト」として残り、他4名が他グループへ移動。ホストが自グループの業務フローを説明し、訪問者がフィードバック・質問を行う。
- **ラウンド2（15分）**: メンバーを入れ替えて再度実施。異なる視点からの気づきを得る。

**期待効果:**
- 他チームの業務プロセスを知ることで、横連携のヒントを得る
- 客観的な視点から自チームの改善ポイントに気づく
- 組織全体の業務理解が深まる

#### フェーズ4: ボトルネック特定・改善策立案（30分）
**前半15分：ボトルネック特定**
- ワールドカフェで得たフィードバックを反映
- 可視化した業務フローから「時間がかかる」「手戻りが多い」「属人化している」箇所を特定
- グループディスカッションで優先順位をつける
- ボトルネックTOP3を選定

**後半15分：改善策立案**
- ボトルネックに対する改善アイデアをブレインストーミング
- 実現可能性と効果を評価
- 改善計画（誰が・いつまでに・何をするか）を策定

#### フェーズ5: 休憩（10分）

#### フェーズ6: 発表会（20分）
- 各グループが改善計画を発表（1グループ3-4分）
- 質疑応答・講師コメント
- ベスト改善アイディア賞の候補選定

**合計時間: 120分**
（フェーズ1:10分 + フェーズ2:20分 + フェーズ3:30分 + フェーズ4:30分 + フェーズ5:10分 + フェーズ6:20分 = 120分）

### 2.4 ワークショップ②詳細設計（90分）

#### テーマ: データ活用による業務改善

**目的:**
- データを用いた論理的な分析手法を体験する
- 数値から改善ポイントを発見する力を養う
- 個人の気づきを言語化・発表する経験を積む

**進行:**
1. **仮想データの提示**（10分）
   - 経理、総務、人事などの業務データ（例：処理時間、エラー率、件数推移）

2. **グループ分析ワーク**（40分）
   - データから課題仮説を立てる
   - 改善のポイントを議論
   - 具体的な施策を考える

3. **個人発表準備**（20分）
   - 各メンバーが「自分が取り組みたい改善策」を1つ選び、1分間スピーチの準備

4. **個人発表**（20分）
   - 1人1分×30名（時間調整により一部代表者のみも可）
   - 「この改善に取り組みたい」という意思表明

## 5. 要件定義

### 5.1 機能要件

#### FR-01: 講義コンテンツ
- **要件**: 管理部門のDX・業務改善に関する30分の講義を提供する
- **内容**:
  - RPA（Robotic Process Automation）の基礎と活用事例
  - AI（生成AI含む）の業務活用シーン
  - クラウドツールによる業務効率化
  - 分科会前段の事前準備としての知識インプット
- **成功基準**: 参加者の70%以上が「DXツールの理解が深まった」と回答

#### FR-02: ワークショップ①「無駄を剥ぐ×業務改革の視点」
- **要件**: 120分で業務フロー可視化→ボトルネック特定→改善策立案を完遂する
- **成功基準**:
  - 全5グループが業務フロー見える化シート（模造紙）を完成
  - 全5グループがボトルネックTOP3を特定
  - 全5グループが改善計画を策定

#### FR-03: ワークショップ②「データ活用」
- **要件**: 90分で仮想データ分析→改善ポイント抽出→個人発表を実施
- **成功基準**:
  - 全グループが分析ワークを完了
  - 参加者の80%以上が個人発表を実施（時間の都合で代表者のみの場合も可）

#### FR-04: ワールドカフェ
- **要件**: 15分×2ラウンドで知識の相互学習を促進
- **成功基準**:
  - 参加者が他グループの業務を理解し、自グループへのフィードバックを得る
  - アンケートで「他チームの視点が参考になった」が70%以上

#### FR-05: ファシリテーション
- **要件**: 講師がグループワークを巡回し、適切なアドバイス・問いかけを行う
- **成功基準**: 各グループ最低3回以上の訪問・支援

---

### 5.2 非機能要件

#### NFR-01: 参加者体験（UX）
- 参加者が「自分ごと」として業務改善に向き合える雰囲気作り
- 心理的安全性を確保し、発言しやすい環境を提供
- 楽しみながら学べる設計（ゲーミフィケーション要素）

#### NFR-02: 時間管理
- タイムテーブル遵守率90%以上（±5分以内の誤差）
- 休憩時間の確保（最低10分）

#### NFR-03: 資料品質
- 誤字脱字ゼロ
- 視認性の高いスライドデザイン（フォントサイズ24pt以上）
- 印刷資料はカラー推奨（ただし予算次第）

#### NFR-04: アクセシビリティ
- 聴覚障害者向け配慮（スライドに文字情報を十分に記載）
- 高齢参加者向け配慮（大きな文字、ゆっくりとした説明）

#### NFR-05: セキュリティ・機密性
- セミナー資料は社外秘扱い
- 参加者の個人情報は適切に管理
- ワークショップで出た業務情報は口外しない旨を事前周知

## 13. NotebookLM活用判断

### 13.1 採用判断

### 13.2 不採用理由

マナテック株式会社は現時点でNotebookLMの導入ステージが低いため、今回のセミナーでは**NotebookLMを使用しない**こととします。

**代替手段:**

**将来的な検討事項:**
マナテック社の業務改善活動が定着し、デジタルツール活用への理解が深まった段階で、NotebookLM導入を再検討することを推奨します。
