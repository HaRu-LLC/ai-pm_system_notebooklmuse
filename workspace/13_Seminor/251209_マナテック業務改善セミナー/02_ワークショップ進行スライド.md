## 模造紙のレイアウトイメージ

**担当者ごとにレーンを分ける → 業務の受け渡しが一目瞭然**

```
┌────────────────────────────────────────────────────────────────┐
│           【業務フロー見える化シート】 グループ名：_______      │
├─────────┬──────────────────────────────────────────────────────┤
│ [担当A] │→【業務1】──────────→【業務3】──────────→             │
│         │  苦労: ______        成果物: ______                  │
├─────────┼──────────────────────────────────────────────────────┤
│ [担当B] │      └→【業務2】 →┘                                  │
│         │         苦労: ______                                 │
├─────────┼──────────────────────────────────────────────────────┤
│ [担当C] │                            └→【業務4】→【完了】      │
│         │                               工夫: ______           │
├─────────┴──────────────────────────────────────────────────────┤
│  📥 インプット欄       │  📤 アウトプット欄                     │
│  （必要な情報・資料）  │  （最終成果物）                        │
```

### ✅ スイムレーンのメリット

- **担当者間の業務の受け渡しが明確**
- **属人化しているポイントが発見しやすい**
- **待ち時間・ボトルネックが視覚的に見える**

# **スライド6: ワークの進め方（重要）**

## 🚀 効率的に進めるコツ

### 1️⃣ **まず全体の流れを決める**
「最初に何をして、最後に何をするか」
→ 大きな流れを模造紙に配置

### 2️⃣ **詳細は後回し**
すべての項目を埋めようとしない
→ 主要な業務から優先的に

### 3️⃣ **全員で考える**
特定の人だけが話さない
→ 1人1業務は必ず発言

### 4️⃣ **完璧を目指さない**
80%の完成度で次へ
→ ワールドカフェで改善

# **スライド11: ワールドカフェとは？**

## 🗣️ ワールドカフェ = 知識の相互学習

### 目的

- **他チームの業務を知る**
- **客観的な視点で自チームを見る**
- **組織全体の業務理解が深まる**

### 仕組み

```
ラウンド1（15分）
  ↓
各グループから1名が「ホスト」として残る
他4名が他グループへ移動
  ↓
ホストが自グループの業務フローを説明
訪問者がフィードバック・質問
  ↓
ラウンド2（15分）
  ↓
メンバーを入れ替えて再度実施
```

# **スライド22: ボトルネックの見つけ方（復習）**

## 🔍 ボトルネックの3つの視点

### 1️⃣ **時間がかかる**
- 「この作業に何時間もかかっている」
- 「待ち時間が長い」
- 「承認が遅い」

### 2️⃣ **手戻りが多い**
- 「やり直しが頻繁に発生する」
- 「ミスが多い」
- 「確認作業が煩雑」

### 3️⃣ **属人化している**
- 「〇〇さんしかできない」
- 「マニュアルがない」
- 「引き継ぎが困難」

## 📝 ステップ2: SPRINGラベルで根本原因を分類

### 苦労点の付箋にSPRINGラベルを書き込む

**マーカーで付箋の角にS/P/R/I/N/Gを記入**

| ラベル | 観点 | 該当する課題例 |
|---|---|---|
| **S** | System（システム） | ツールが古い、使いにくい、セキュリティ制約 |
| **P** | People（人材） | スキル不足、抵抗勢力、教育機会がない |
| **R** | Resources（データ） | 情報が散在、資料が見つからない |
| **I** | Integration（連携） | 部署間の情報共有不足、縦割り |
| **N** | Navigation（推進） | リーダー不在、時間が確保できない |
| **G** | Goals（目標） | ゴール不明確、優先順位がない |

```
┌─────────────────────────────────┐
│ 苦労点の付箋の例                │
│ ┌─────────────────────────┐    │
│ │ [S] ← SPRINGラベル     │    │
│ │ 手入力に時間がかかる   │    │
│ │ Excelが古くてマクロ    │    │
│ │ が使えない             │    │
│ └─────────────────────────┘    │
└─────────────────────────────────┘
```

## 📝 ステップ3: ボトルネックTOP3を選定

### 優先順位の判断基準

- **影響度**: 業務全体に与える影響は大きいか？
- **改善可能性**: 自分たちで改善できるか？
- **緊急性**: すぐに取り組むべきか？

### 💡 SPRINGラベルの集計

**最も多いラベル = 優先的に改善すべき領域**

例）S:2個、P:5個、R:3個 → **P（人材）が最大の課題**

→ **グループで議論してTOP3を決定**

# **スライド27: 改善策立案の4つの視点**

## 💡 改善の4つの方向性（ECRS）

### 1️⃣ **なくす（Eliminate）**
- 「この作業は本当に必要か？」
- 不要な承認、重複作業の削減

### 2️⃣ **減らす（Reduce）**
- 「もっと簡単にできないか？」
- 入力項目の削減、頻度の見直し

### 3️⃣ **変える（Change）**
- 「別の方法でできないか？」
- ツール導入、プロセスの変更

### 4️⃣ **組み合わせる（Combine）**
- 「まとめてできないか？」
- 業務の統合、一括処理
