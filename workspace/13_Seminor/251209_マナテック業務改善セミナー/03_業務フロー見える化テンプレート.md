# 業務フロー見える化テンプレート
**マナテック株式会社 管理本部 業務改善セミナー**

---

## テンプレートの使い方

このテンプレートは、**模造紙＋付箋**で業務フローを見える化するためのガイドです。
以下の手順に従って、業務プロセスを可視化してください。

---

## 📋 業務フロー見える化の5項目（優先度順）

| 優先 | 項目 | 記載内容 | 例 |
|---|---|---|---|
| ★★★ | ① **業務名** | タスクの名称 | 「月次決算」「勤怠集計」 |
| ★★★ | ② **担当者** | 実施者（役割） | 「経理担当Aさん」「人事チーム全員」 |
| ★★☆ | ③ **出力（成果物）** | 何ができあがるか | 「月次レポート」「給与明細」 |
| ★★★ | ④ **苦労している点** | 課題・困りごと | 「手入力に時間がかかる」「ミスが多い」 |
| ★☆☆ | ⑤ **工夫している点** | ノウハウ・ベストプラクティス | 「チェックリストを使っている」 |

> **💡 フェーズ4で苦労点をSPRINGの6観点で分類します**

## 🎨 模造紙のレイアウト例

### 🏆 推奨：スイムレーン型（担当者別）

**担当者ごとにレーンを分ける → 業務の受け渡しが一目瞭然**

```
┌────────────────────────────────────────────────────────────────┐
│           【業務フロー見える化シート】 グループ名：_______      │
├─────────┬──────────────────────────────────────────────────────┤
│ [担当A] │→【業務1】──────────→【業務3】──────────→             │
│         │  苦労: ______        成果物: ______                  │
├─────────┼──────────────────────────────────────────────────────┤
│ [担当B] │      └→【業務2】 →┘                                  │
│         │         苦労: ______                                 │
├─────────┼──────────────────────────────────────────────────────┤
│ [担当C] │                            └→【業務4】→【完了】      │
│         │                               工夫: ______           │
├─────────┴──────────────────────────────────────────────────────┤
│  📥 インプット欄       │  📤 アウトプット欄                     │
│  （必要な情報・資料）  │  （最終成果物）                        │
└────────────────────────────────────────────────────────────────┘
```

**メリット:**
- ✅ 担当者間の業務の受け渡しが明確
- ✅ 属人化しているポイントが発見しやすい
- ✅ 待ち時間・ボトルネックが視覚的に見える

## 🚀 作成の手順

### Step 1: 業務名を書き出す（5分）

- **各メンバーが自分の担当業務を付箋に書く**
- **1業務1枚の付箋**
- **できるだけ具体的に**

**例:**
- ❌ 「経理業務」（抽象的）
- ✅ 「月次決算データ入力」（具体的）

---

### Step 2: 業務の流れを並べる（10分）

- **付箋を模造紙に貼る**
- **時系列順に並べる**
- **最初の業務→最後の業務の流れを作る**

**ポイント:**
- 完璧な順序でなくてOK
- 大まかな流れを作ることが重要

---

### Step 3: 詳細情報を追記する（5分）

- **インプット・アウトプットを書く（青色・ピンク色の付箋）**
- **担当者を明記する**
- **苦労点・工夫点を書く（緑色の付箋）**

**ポイント:**
- すべての項目を埋める必要はない
- 重要な業務から優先的に

## 🔍 ボトルネックの見つけ方（フェーズ4で実施）

業務フローが見える化できたら、**SPRINGラベル方式**で根本原因を分類してください。

### SPRINGラベル方式

**苦労点の付箋にマーカーでS/P/R/I/N/Gを書き込む**

| ラベル | 観点 | 該当する課題例 |
|---|---|---|
| **S** | System（システム） | ツールが古い、使いにくい、セキュリティ制約 |
| **P** | People（人材） | スキル不足、抵抗勢力、教育機会がない |
| **R** | Resources（データ） | 情報が散在、資料が見つからない |
| **I** | Integration（連携） | 部署間の情報共有不足、縦割り |
| **N** | Navigation（推進） | リーダー不在、時間が確保できない |
| **G** | Goals（目標） | ゴール不明確、優先順位がない |

### 💡 最も多いラベル = 優先的に改善すべき領域

例）S:2個、P:5個、R:3個 → **P（人材）が最大の課題**

---

### 従来の3視点も参考に

| 視点 | チェック項目 |
|---|---|
| **時間がかかる** | 何時間もかかる、待ち時間が長い、承認が遅い |
| **手戻りが多い** | やり直し多発、ミスが多い、確認作業が煩雑 |
| **属人化している** | 〇〇さんしかできない、マニュアルがない |

---

### 視点2: 手戻りが多い

- [ ] やり直しが頻繁に発生する
- [ ] ミスが多い
- [ ] 確認作業が煩雑

**マーカー色: 青色**

---

### 視点3: 属人化している

- [ ] 〇〇さんしかできない
- [ ] マニュアルがない
- [ ] 引き継ぎが困難

**マーカー色: 緑色**

## 💡 改善アイデアの出し方

ボトルネックが見つかったら、以下の4つの視点で改善アイデアを考えてください。

### ECRS（イクルス）

| 視点 | 英語 | 問いかけ | 例 |
|---|---|---|---|
| **E** | Eliminate | 「なくせないか？」 | 不要な承認を削減 |
| **C** | Combine | 「まとめられないか？」 | 複数の業務を統合 |
| **R** | Rearrange | 「順序を変えられないか？」 | 並行処理に変更 |
| **S** | Simplify | 「簡素化できないか？」 | 入力項目を削減 |

## 🌸 SPRING診断チェックリスト（参考）

業務改善の準備度を自己診断してください。

### S: System（システム・環境）
- [ ] 業務を効率化するツール・システムがある
- [ ] 既存システムは使いやすい
- [ ] セキュリティ・ガイドラインが整備されている

### P: People（人材・スキル）
- [ ] 業務改善に取り組む人材がいる
- [ ] 変革への抵抗は少ない
- [ ] 新しいツールを学ぶ意欲がある

### R: Resources（データ・情報）
- [ ] 業務データは可視化されている
- [ ] 情報は整理・共有されている
- [ ] 過去の資料はすぐに見つかる

### I: Integration（連携・協力）
- [ ] 部署間で情報共有できている
- [ ] 他部署の成功事例を学んでいる
- [ ] 横連携がスムーズ

### N: Navigation（推進体制）
- [ ] 改善を推進するリーダーがいる
- [ ] 経営層のサポートがある
- [ ] 改善活動の時間が確保されている

### G: Goals（目的・目標）
- [ ] 何のために改善するのか明確
- [ ] 改善の優先順位がついている
- [ ] 成功の基準が定義されている

## 📌 よくある質問（FAQ）

### Q1: 業務が多すぎて、すべて書き出せません

**A**: すべて書き出す必要はありません。**主要な業務**（時間がかかる、重要度が高い）から優先的に可視化してください。

---

### Q2: 付箋の色が足りなくなりました

**A**: 同じ色を使ってもOKです。重要なのは**業務の流れを可視化すること**です。色分けはあくまで補助的なツールです。

---

### Q3: グループ内で意見が分かれています

**A**: 完璧な答えを出す必要はありません。**多数決**または**時間がかかる方を採用**して、とりあえず進めてください。

---

### Q4: ワールドカフェで何を話せばいいですか？

**A**: 以下の3つを中心に話してください：
1. **自チームの業務フローの説明**
2. **苦労している点の共有**
3. **他チームへの質問・フィードバック**

---

### Q5: 改善アイデアが思いつきません

**A**: 以下の問いかけを試してください：
- 「この業務は本当に必要か？」（なくす）
- 「もっと簡単にできないか？」（簡素化）
- 「別の方法でできないか？」（変える）
- 「まとめてできないか？」（統合）
